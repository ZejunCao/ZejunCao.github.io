2025-01-23｜UCSC,AdobeResearch
http ://arxiv.org/abs/2501.13896v1https ://gui-bee.github.io
本文提出了一种名为GUI-Bee的自主探索代理，旨在将图形用户界面（GUI）动作定位模型与新环境对齐。当前的GUI动作定位模型通常依赖于大规模的训练数据，但这些数据往往无法涵盖所有可能的GUI环境，导致模型在新环境中的表现不佳。GUI-Bee通过自主探索来收集特定环境的数据，并利用这些数据对模型进行持续的微调，从而提升其在未见环境中的性能。此外，研究中引入了一种新颖的Q值激励上下文强化学习（Q-ICRL）方法，以优化数据收集的效率和质量，从而有效地增强模型的适应能力。
本研究的方法主要包括以下几个步骤：
自主探索：GUI-Bee代理从预定义的初始GUI屏幕开始，通过预测和执行动作与GUI进行交互，收集在每一步操作前后的屏幕数据，从而构建一个探索图。
Q值激励上下文强化学习（Q-ICRL）：该方法通过在探索过程中利用Q值函数来评估每个动作的潜在收益，帮助代理在面对无效或重复动作时选择最优的行动策略。Q-ICRL是训练无关的，依赖于记忆机制来动态调整动作选择。
数据微调：通过使用GUI-Bee收集到的数据，对现有的GUI动作定位模型进行微调，以提升其在新环境中的表现。微调过程包括整合来自不同配置（如仅视觉和视觉+无障碍）的数据，以增强模型的环境适应性。
在实验部分，研究者使用NovelScreenSpot基准测试来评估GUI-Bee收集的数据如何帮助对齐GUI动作定位模型。实验分为几个步骤：
环境选择：选取五个未见过的GUI环境进行探索，确保涵盖多样化的界面设计和功能。
性能评估：通过比较微调前后模型在NovelScreenSpot基准测试中的表现，量化模型在新环境中的改进幅度。
探索效率评估：与其他基线代理进行比较，评估GUI-Bee在探索和数据生成过程中的效率，确保其在相同条件下具有更广泛的探索覆盖率。
在这项研究中，GUI-Bee就像一个聪明的机器人，它可以在各种软件界面中自由探索。首先，这个机器人会从一个界面开始，尝试点击按钮或者选择菜单，以此来获取新的信息和图片。然后，它会使用一种特殊的数学方法（Q-ICRL）来判断哪些动作会带来最好的结果，这样就能避免重复做一些没有意义的事情。最后，当它收集到足够的数据后，这些信息会被用来帮助其他程序更好地理解和适应新的界面。这就好比一个学生在不同的课堂上学习，通过不断的练习和反馈，最终能够在任何课堂上表现得更好。
2025-01-23｜Stanford,USTC
http ://arxiv.org/abs/2501.13919v1https ://ruili33.github.io/tpo_website
在长视频理解领域，尽管视频大规模多模态模型（video-LMMs）取得了显著进展，但实现有效的时间锚定仍然是一个挑战。为了解决这一问题，本文提出了一种新颖的后训练框架——时间偏好优化（TemporalPreferenceOptimization,TPO）。TPO旨在通过偏好学习增强视频-LMMs的时间锚定能力。该方法采用自我训练的方式，利用局部和综合时间偏好数据集，帮助模型区分时间响应的准确性，从而提高对视频内容的理解。经过在多个长视频理解基准上进行的广泛实验，TPO展示了其在提升视频-LMMs时间理解能力方面的有效性，尤其是在需要细粒度或长上下文时间锚定的任务中。
TPO方法主要包括以下几个步骤：
数据生成：TPO通过生成时间偏好数据集来优化视频-LMMs的时间锚定能力。数据集由四元组（视频V，查询q，优选响应r+，不优选响应r-）构成。
局部时间偏好数据生成：针对视频的特定片段生成问题，并使用图像语言模型生成每帧的描述，进而生成与特定片段相关的查询和优选响应。
综合时间偏好数据生成：从整个视频中采样帧，创建更广泛的查询，确保模型能够识别和理解视频中的重要信息。
偏好优化：使用直接偏好优化（DirectPreferenceOptimization,DPO）算法，通过优化模型的响应概率，使其更倾向于生成优选响应。
后处理：对生成的偏好数据进行过滤，确保数据的质量和有效性，以提升模型的训练效果。
在实验中，TPO在三个广泛认可的长视频理解基准上进行了评估：LongVideoBench、MLVU和Video-MME。实验结果表明，TPO显著提升了视频-LMMs的时间理解能力，LongVA-TPO和LLaVA-Video-TPO在多个基准上超越了现有的最先进模型。具体来说，LongVA-TPO在LongVideoBench、MLVU和Video-MME基准上分别提高了2.9%、3.1%和2.5%的性能。此外，TPO还展示了其在处理长输入序列时的优势，能够有效地定位和理解视频中的关键信息。这些结果表明，TPO是一种可扩展且高效的解决方案，能够推动长视频理解领域的发展。
TPO的工作原理可以简单理解为一个“学习偏好”的过程。首先，我们从视频中提取出一些短片段，并为这些片段设计具体的问题。然后，我们使用先进的模型来生成这些片段的描述，以帮助我们理解它们的内容。接着，我们会创建两个类型的答案：一个是“优选答案”，它是与视频片段内容紧密相关的；另一个是“不优选答案”，它则是从不相关的片段中生成的。这两个答案帮助模型学习到什么样的回答是最合适的。最后，模型通过不断优化这些答案，逐步提升其在理解视频内容时的准确性和效率。通过这种方式，TPO能够让视频理解模型在处理复杂视频时表现得更加出色。
2025-01-23｜DLUT,Harvard｜ICLR2025
http ://arxiv.org/abs/2501.13468v1https ://github.com/hmxiong/StreamChat
近年来，随着大型语言模型（LLMs）的发展，视频理解领域也取得了显著进展，尤其是视频-LLMs的出现，使得多模态学习得以推进。然而，现有的视频理解模型在处理长视频序列、支持多轮对话和适应动态场景方面仍面临挑战。为了解决这些问题，研究者们提出了STREAMCHAT，一个无训练的框架，专注于流媒体视频推理和对话交互。该框架利用一种新颖的层次记忆系统，能够高效处理和压缩长视频序列中的特征，从而实现实时的多轮对话。此外，STREAMCHAT还引入了一种并行系统调度策略，显著提高了处理速度并降低了延迟，确保了在实际应用中的稳健表现。为了评估流媒体视频理解的效果，研究者们还推出了STREAMBENCH，一个多样化的基准测试，涵盖了多种媒体类型和交互场景。
STREAMCHAT的设计包含三个关键创新：首先是无训练适应性，使其能够高效处理各种类型和长度的视频，而无需资源密集型的训练。这一特性使得STREAMCHAT适用于在线和离线视频处理，且在多种场景中保持稳定性能。其次是层次记忆存储，能够管理和压缩长序列中的视频信息。该系统结合了短期记忆以跟踪当前事件，长期记忆以保留过去事件的压缩形式，以及对话记忆以维持对话历史，从而确保对话理解的连续性和一致性。最后，优化的系统调度策略通过并行处理任务提高了模型推理效率，具体包括选择性帧堆叠、记忆形成和上下文摘要三个线程，分别用于识别和去除冗余帧、更新记忆信息和实时生成响应。
在评估STREAMCHAT的性能时，研究者们使用了现有的基准测试并发现其存在的两大主要缺陷：一是视频内容短且单调，无法捕捉到现实世界流媒体的复杂性；二是问题设计过于简单，未能测试模型在多轮对话或复杂推理方面的能力。为了解决这些问题，STREAMBENCH被引入，旨在模拟真实世界的交互。该基准测试包括来自四个主要领域和十六个子类的视频，涵盖了丰富多样的内容，并设计了六种不同类型的问题，以探测模型的推理能力。通过严格的人工筛选和多模态语言模型的辅助，最终构建了一个包含306个视频的高质量数据集，确保了基准测试的有效性和可靠性。
在STREAMCHAT的设计中，研究者们采用了一种类似于人类记忆的方式来处理视频信息。首先，它能够在没有额外训练的情况下，快速适应不同类型和长度的视频，这就像我们看视频时，不需要重新学习每个视频的内容一样。其次，STREAMCHAT使用了一种层次化的记忆结构，分为短期和长期记忆。短期记忆帮助它记住刚刚发生的事情，而长期记忆则保留了以前发生的重要信息，确保在对话中能够保持连贯性。最后，系统通过并行处理的方式来提高效率，就像我们在做多件事情时能够同时进行一样，使得它在回答问题时迅速而准确。这些创新使得STREAMCHAT在处理实时视频时表现出色，能够适应复杂的对话场景。
2025-01-23｜NTUS-Lab,CMU｜🟡
http ://arxiv.org/abs/2501.13826v1https ://videommmu.github.io/
本研究提出了一种新的视频基准测试工具——Video-MMMU，旨在评估大型多模态模型（LMMs）从多学科专业视频中获取知识的能力。该工具基于人类学习知识的三个认知阶段：感知、理解和适应。研究团队收集了300个专家级视频和900个手工标注的问题，涵盖艺术、商业、科学、医学、人文学科和工程等六个学科。通过设计与这三个认知阶段相对应的问答对，Video-MMMU为评估模型的知识获取能力提供了系统化的方法。此外，研究还提出了一种新的知识获取度量指标，Δ，量化观看视频后的表现提升，突显出当前模型在知识适应能力方面与人类的显著差距。
Video-MMMU的设计包括以下几个关键部分：
视频收集：团队通过对30个学科的大学课程进行深入分析，筛选出450个基础评估主题，最终选定300个教育视频。这些视频涵盖概念介绍和问题解决两种类型，确保内容丰富且具有挑战性。
问答设计：每个视频配备与认知阶段相对应的三个问题，分别评估感知（提取关键信息）、理解（掌握概念）和适应（将知识应用于新场景）的能力。问题形式包括多选题、概念理解和案例分析等。
知识获取评估：通过Δ指标，量化模型在观看视频后在实际考试问题上的表现提升，以评估其知识获取能力。这一方法允许对不同难度基线的表现进行比较，确保评估的公正性。
实验部分涉及对多种开源和专有LMMs的评估，包括LLaVA、Claude等。研究团队通过设置基线，使用一系列视频和问题对模型进行测试，分析其在感知、理解和适应三大认知阶段的表现。实验结果显示，尽管模型在感知任务上表现较好，但在理解和适应任务中性能显著下降，尤其是在适应任务中，许多模型的得分低于50%。此外，研究还比较了不同学科间的表现差异，发现模型在概念呈现较为简单的艺术和人文学科中表现较好，而在需要复杂计算的科学和工程领域表现不佳。
在Video-MMMU的评估方法中，我们首先收集了大量的教育视频，确保这些视频能够传授重要的知识。接着，我们为每个视频设计了三个类型的问题，帮助我们了解学习者在观看后能否提取信息、理解概念和将学到的知识应用到新的场景中。通过观看视频后，学习者的表现提升情况被量化成一个指标，称为Δ。这就像是我们在考试前复习后，考试成绩的提高程度。最终，通过对不同模型的测试，我们发现虽然它们在提取信息时表现不错，但在理解和应用知识时却常常遇到困难，这显示出模型在学习和应用知识方面与人类之间的差距。
