271-B1｜面向GPT-4o的实时视觉与语音交互，多模态慢思维系统，面向长视频理解的大规模基准与数据集，图像描述生成

主编｜庄才林（CailinZhuang）技术支持｜胡耀淇（YaoqiHu）发布日期｜2025年1月6日｜周一
2025-01-03｜NJU,TencentYoutuLab,XMU,CASIA｜⭐️
http ://arxiv.org/abs/2501.01957v1https ://github.com/VITA-MLLM/VITA
VITA-1.5是一种多模态大语言模型（MLLM），旨在实现实时的视觉与语音交互。尽管近年来的研究主要集中在视觉和文本的结合上，但语音在多模态对话系统中的重要性日益凸显。VITA-1.5通过一种精心设计的三阶段训练方法，逐步培养模型理解视觉和语音信息的能力，从而实现流畅的视觉与语音交互。该模型不仅保持了强大的视觉-语言能力，还能高效地进行语音对话，无需依赖单独的自动语音识别（ASR）和文本转语音（TTS）模块，从而显著加快多模态的端到端响应速度。通过与先进模型的基准比较，VITA-1.5在图像、视频和语音任务上展示了其强大的视觉和语音能力，实现了近实时的互动效果。
VITA-1.5的训练方法分为三个阶段，旨在逐步引入不同的模态以减轻训练中的冲突。具体步骤如下：
视觉-语言训练：
视觉对齐：使用部分描述性数据训练视觉适配器，冻结其他模块，确保模型初步对视觉模态的理解。
视觉理解：利用全部描述性数据训练，增强模型对图像内容的理解和生成自然语言描述的能力。
视觉指令调优：结合视觉问答数据，提升模型在视觉理解和指令遵循方面的能力。
音频输入调优：
音频对齐：使用大量语音-转录对数据，训练音频编码器，使其能够提取语音特征并映射到文本表示。
音频指令调优：引入语音问题与文本答案的结合，增强模型对语音输入的理解能力。
音频输出调优：
编解码器训练：训练一个编解码器模型，将语音映射为离散的语音令牌，并在推理阶段仅使用解码器。
非自回归与自回归解码器训练：基于文本-语音配对数据，训练解码器生成高质量的语音输出。
VITA-1.5在多个基准测试上进行了广泛评估，包括图像理解、视频理解和语音识别任务。实验结果表明，VITA-1.5在视觉-语言能力上与领先的开源模型表现相当，甚至在某些任务上超越了闭源模型，如GPT-4o。具体而言，在图像理解方面，VITA-1.5的表现与多种开源模型相当，而在视频理解和自动语音识别（ASR）任务中，VITA-1.5显示出强大的性能，尤其是在中文和英文的ASR任务中，表现优于专门的语音模型。
VITA-1.5的训练方法可以想象成一个逐步学习的过程。首先，模型学习如何理解图片和视频，就像一个小孩通过看图画书来理解故事。接下来，它开始听语音并学习将声音转换成文字，这就像在课堂上听老师讲课。最后，模型还学会了如何把文字变成流利的语音，就像一个人学会了朗读课文。通过这种逐步的学习方式，VITA-1.5能够同时处理视觉和语音信息，让人机交互变得更加自然和流畅。
2025-01-03｜RUC,BaichuanAI,BAAI｜🟡
http ://arxiv.org/abs/2501.01904v1https ://github
近年来，基于大型语言模型（LLMs）的慢思维推理系统引起了广泛关注，这些系统通过延长推理时间来提高性能。随着多模态大型语言模型（MLLMs）的发展，如何将慢思维能力有效地应用于处理更复杂的数据语义成为一个重要课题。本文提出了一种简单的方法，通过对一个强大的MLLM进行微调，使用少量的长文本推理数据，构建了多模态慢思维系统“Virgo”。研究发现，长文本推理过程能够有效地转移至MLLMs，且文本推理数据在激发慢思维能力方面的效果优于视觉推理数据。尽管本研究仍处于初步阶段，但结果表明，慢思维能力与语言模型组件密切相关，能够在不同模态或领域间转移。
本研究探讨了两种简单的适应方法来增强MLLM的慢思维能力：
从文本指令中转移能力：通过微调MLLM，使其能够利用长文本推理数据。我们从先前研究中收集了约5000个长思维指令实例，主要来自数学领域，以确保包含更复杂的问题。这些指令分为思维过程和最终解决方案两个部分，便于模型学习。
从慢思维MLLM中蒸馏多模态长思维数据：通过从现有的慢思维MLLM（如QVQ）中提取视觉问题和答案，生成视觉长思维数据。这一过程旨在比较文本和多模态长思维数据的微调效果，并探讨两者结合的潜力。
在具体实施中，选择了具有强大多模态能力的MLLM作为基础模型，通过不同的数据集进行微调，以评估其推理性能。
为验证方法的有效性，我们在四个具有挑战性的基准上进行了广泛的实验：MathVerse、MathVision、OlympiadBench和MMMU。实验结果显示，经过仅5000个文本长思维指令微调后，Virgo在多个基准上的表现达到了与商业推理系统相当的水平。特别是在MathVision和OlympiadBench上，Virgo的准确率分别达到38.4%和29.3%。此外，实验还表明，文本推理指令通常比多模态推理数据更有效地激发MLLM的慢思维能力。通过对不同难度样本的细致分析，我们发现，较难任务更能从长思维推理中获益，而增加指令数据的数量也能提升整体性能。
在这项研究中，我们想要让大型语言模型（MLLM）变得更聪明，能够像人类一样进行深思熟虑的推理。我们采用了两种简单的方法。首先，我们从以前的研究中收集了一些长文本推理的例子，主要是数学问题。这些例子帮助模型学习如何进行长时间的思考。其次，我们还从现有的慢思维模型中提取了一些视觉问题，生成了与图片相关的推理数据。通过这些数据，我们对模型进行了微调，看看它在解决问题时的表现如何。最后，我们在多个复杂问题的测试中发现，使用文本指令的效果往往比使用视觉数据更好，说明让模型通过长文本进行思考是一个有效的策略。
2025-01-03｜TikTok,NTU,SWJTU,GreatBayU,SZU
http ://arxiv.org/abs/2501.01645v1
在本研究中，我们提出了HLV-1K，这是一个专门为长视频理解而设计的大规模基准数据集。HLV-1K包含1009段超过一小时的长视频，配有14,847对高质量的问答（QA）和多选问答（MCQA）标注。这些标注不仅涵盖了时间感知的查询，还包括多种层次的推理任务，如帧级、事件内、跨事件以及长期推理任务。该基准旨在评估多模态大语言模型（MM-LLMs）在处理复杂长视频内容时的能力，并推动长视频理解领域的研究和应用。
HLV-1K的构建过程包括以下几个关键步骤：
数据收集：我们从公共资源中收集了超过30分钟的长视频，最终筛选出1009个高质量视频，涵盖各种主题，如娱乐、科技和旅行。
数据标注：
帧提取：从每个视频中提取关键帧，压缩到每秒一帧，以减少冗余信息。
帧标注：使用GPT-4o对每帧进行描述，结合YOLO进行对象识别，以确保描述的准确性和全面性。
事件标注：通过滑动窗口方法将连续帧的描述合并为事件描述，生成时间感知的问答对。
数据过滤：对标注进行严格审查，去除低质量的问答对，确保所有问题和答案的准确性和一致性。
在实验部分，我们使用现有的最先进方法对HLV-1K进行评估。我们测试了多种商业和开源的多模态语言模型，包括GPT-4o和LLaVA-Video等。实验结果表明，不同模型在各种任务上的表现存在显著差异，尤其是在处理复杂的时间和空间推理任务时。我们的评估揭示了当前模型的优缺点，并为未来的研究提供了重要的见解，以改进长视频理解的能力。
在HLV-1K的构建过程中，我们采用了一种系统的方法来确保数据的质量和多样性。首先，我们从互联网上收集了许多长视频，确保每个视频都超过30分钟。接着，我们对视频进行了处理，提取出每秒一帧的关键画面，这样可以减少重复和冗余的信息。然后，通过先进的人工智能工具，我们为每一帧和每个事件编写详细的描述，确保这些描述不仅准确，还能反映出视频的主要内容。最后，我们对所有的问答对进行了仔细的审查，确保它们的格式正确且与视频内容相符。这样，我们就能创建一个高质量的数据集，帮助研究人员更好地理解长视频的复杂性。
2025-01-03｜PKU
http ://arxiv.org/abs/2501.01834v1
本研究提出了一种名为MoColl的图像描述生成框架，该框架通过结合特定领域的视觉分析与广泛的上下文知识，旨在克服现有模型在图像描述任务中的局限性。图像描述是计算机视觉与自然语言处理交叉领域的重要任务，尤其在放射学报告生成等复杂应用中显得尤为重要。传统方法往往依赖于专门的编码-解码模型，虽然在特定领域表现良好，但缺乏广泛的适应能力。而基于大语言模型的视觉-语言模型（VLM）虽然具备较强的通用性，但在处理特定领域任务时仍面临挑战。为此，MoColl框架通过将复杂的图像描述任务分解为一系列互相关联的问答子任务，结合了视觉问答（VQA）模型与大型语言模型（LLM）代理，旨在实现这两者的有效协作。
MoColl框架的核心在于其模型协作和代理引导的调优过程。具体而言，方法分为两个主要步骤：
模型协作框架：该框架将图像描述过程分解为问答步骤和描述生成步骤。在问答步骤中，代理基于先前的问答互动向VQA模型提出问题，VQA模型根据输入图像提供答案。此过程持续进行，直到收集到足够的信息或满足停止条件。
代理引导的调优过程：该过程包含两个阶段。第一阶段是“热身阶段”，通过对图像特征与文本嵌入的对齐来准备模型。第二阶段是“代理引导调优”，在这一阶段，代理生成大量合成的问答对，并根据真实的描述过滤出高质量的数据，以进一步精细化VQA模型的表现。
在实验部分，我们使用了两个广泛使用的数据集：IU-Xray和MIMIC-CXR，以验证所提出方法的有效性。实验结果表明，MoColl框架在放射学报告生成任务中显著优于现有的基线方法。通过在特定领域进行训练的VQA模型与LLM代理的结合，MoColl不仅提高了生成描述的准确性，还增强了模型的适应性。此外，我们还进行了消融研究，以评估不同选择策略和数据集大小对模型性能的影响，结果显示，代理引导的选择策略在提高合成数据质量方面表现出色。
在MoColl框架中，图像描述的生成过程可以想象成一个问答游戏。首先，一个智能代理（类似于助手）会根据图像内容向一个专门的视觉问答模型提问。这个模型就像一个专家，它会根据图像内容回答这些问题。这个过程会反复进行，直到代理收集到足够的信息。然后，代理会将这些问答组合成一段完整的描述。为了让这个专家更聪明，代理还会定期对它进行训练，使用从实际数据中生成的高质量问题和答案来提升它的表现。这样，MoColl框架就能有效地结合具体领域的知识和更广泛的背景信息，生成准确且有用的图像描述。
