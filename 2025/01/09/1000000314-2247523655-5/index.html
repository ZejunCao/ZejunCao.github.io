270-B1｜多模态视频理解系列；多模态3D理解系列

主编｜庄才林（CailinZhuang）技术支持｜胡耀淇（YaoqiHu）发布日期｜2025年1月3日｜周五
2024-12-31｜ZJU,AlibabaDAMOAcademy｜🟡
http ://arxiv.org/abs/2501.00599v1https ://damo-nlp-sg.github.io/VideoRefer/
在视频理解领域，视频大型语言模型（VideoLLMs）展现了显著的潜力，但在捕捉细粒度的空间和时间细节方面仍面临挑战。现有的模型多聚焦于整体场景理解，缺乏高质量的对象级视频指令数据和全面的基准测试，限制了其发展。为了解决这些问题，本文提出了VideoReferSuite，旨在提升VideoLLM在视频中的空间-时间对象理解能力。该套件通过构建一个大规模的对象级视频指令数据集VideoRefer-700K、开发一个具备灵活空间-时间对象编码的VideoRefer模型，以及创建一个全面评估模型性能的基准VideoRefer-Bench，来实现对视频内容的精细理解。
VideoReferSuite的核心方法包括三个主要方面：
数据集：构建VideoRefer-700K数据集，利用多代理数据引擎精心策划大规模高质量对象级视频指令数据。该数据引擎通过多个专家模型的协作，生成涵盖基本问题、复杂推理和未来预测的多轮问答对。
模型：引入VideoRefer模型，采用灵活的空间-时间对象编码器，结合空间标记提取器和自适应时间标记合并模块，能够在任意时间戳实现对用户定义区域的精细感知和推理。
基准测试：开发VideoRefer-Bench，分为描述生成和多项选择问答两个子基准，全面评估模型在不同时间戳和对象上的性能，确保对空间-时间理解能力的全面考量。
在实验部分，研究团队利用VideoRefer-Bench对模型进行了详尽的性能评估。通过与多种通用和专业模型的比较，VideoRefer在单帧和多帧模式下均展现出卓越的性能，尤其在对象对应、外观描述和时间描述等方面。实验结果显示，VideoRefer在各种视频引用任务和一般视频理解任务中均优于现有方法，证明了其在空间-时间视频理解中的有效性和先进性。
在VideoReferSuite中，研究者们采用了一种系统化的方法来提升视频理解的能力。首先，他们创建了一个丰富的数据集，包含了大量关于视频中对象的详细描述和问答。这些数据是通过多个智能系统的合作生成的，确保了信息的准确性和多样性。接着，他们开发了一个新的模型，这个模型能够灵活地处理视频中的不同场景，识别特定对象，并理解它们在时间上的变化。最后，他们设计了一套评估工具，能够全面测试模型的表现，确保它能够在实际应用中有效工作。通过这些方法，VideoReferSuite大大提高了我们对视频内容的理解能力。
2025-01-02｜PrincetonU,SalesforceResearch
http ://arxiv.org/abs/2501.01426v1https ://tylerzhu.com/merv/
随着大型语言模型（LLMs）的发展，视频语言模型（VideoLLMs）也逐渐展现出其强大的推理能力。然而，现有的视频语言模型通常依赖于单一的视觉编码器进行视觉处理，这限制了它们所能传达的视觉信息的多样性和深度。针对这一问题，本文提出了一种新的方法，称为MERV（多编码器视频表示），通过结合多个冻结的视觉编码器来创建视频的统一表示。这种方法能够为视频语言模型提供更全面的专业视觉知识，使其在处理开放式和多选视频理解问题时表现优越。实验结果表明，MERV在多个视频理解基准测试中，准确率比现有的最佳模型提高了3.7%，并且在零-shot感知测试中也取得了显著的进步。
MERV的设计围绕三个关键方面展开：编码器选择、特征融合和实现效率。首先，在编码器选择上，MERV使用了四种不同类型的编码器，包括空间专家（DINOv2）、时间专家（ViViT）、图像-语言专家（SigLIP）和视频-语言专家（LanguageBind）。这些编码器各自具备不同的优势，能够补充彼此的不足。其次，在特征融合方面，MERV采用了一种时空对齐的表示形式，通过交叉注意力机制将多个编码器的特征进行有效融合，以便生成一个统一的视觉嵌入。最后，为了提高实现效率，MERV通过并行处理来减少计算开销，同时保持模型的高效训练。
在实验部分，MERV的性能通过在多个视频理解基准上进行评估来验证。实验使用的基准包括MSVD-QA、MSRVTT-QA和ActivityNet-QA等。结果显示，MERV在这些任务中表现优于以往的单一编码器模型，尤其是在复杂的多轮推理和视觉-语言对齐任务中，MERV的表现显著提升。此外，通过与其他最新的视频语言模型进行比较，MERV在多个指标上都取得了更好的结果，表明其在视频理解领域的有效性和潜力。
MERV的工作原理可以简单理解为一个团队合作的过程。想象一下，你需要完成一个复杂的任务，但单靠一个人可能无法做到最好。MERV就像是一个团队，由四个不同的专家组成：一个擅长理解静态图像，一个擅长处理视频中的动作，还有两个分别擅长将图像和文字结合在一起。每个专家都有自己的强项和弱点。通过让这四个专家一起工作，MERV能够更全面地理解视频内容。具体来说，MERV首先从每个专家那里获取信息，然后将这些信息进行整合，形成一个更完整的理解。这种方法不仅提高了准确性，还让视频语言模型能够更好地回答各种问题。
2024-12-31｜ShanghaiAILab,NJU,CASSIAT
http ://arxiv.org/abs/2501.00574v1https ://github.com/OpenGVLab/VideoChat-Flash
本研究提出了一种名为VideoChat-Flash的多模态长视频建模系统，旨在解决当前大型语言模型（MLLM）在处理极长视频时面临的挑战。尽管已有技术在短视频理解上取得了一定进展，但在长视频的上下文理解和信息保留方面仍然存在困难。为此，本文引入了一种称为层次化视觉令牌压缩（HiCo）的方法，旨在通过压缩长视频中的视觉信息来提高模型的效率和有效性。VideoChat-Flash通过多阶段的短到长学习策略和丰富的长视频数据集（LongVid）进行训练，从而实现了对长视频内容的有效处理，显著提高了模型在长视频理解任务中的表现。
VideoChat-Flash的核心创新在于其层次化压缩方法（HiCo），该方法通过以下几个步骤实现高效的长视频建模：
动态视频采样：采用双重采样策略，短视频采用密集帧采样（如15帧每秒），而长视频则使用稀疏帧采样（如1帧每秒），以便于捕捉细节和整体事件理解。
编码与压缩：将长视频分割为多个短片段，并对每个片段进行视觉编码。通过视觉编码器提取的视觉令牌会被压缩，减少输入到大语言模型（LLM）的令牌数量。
视频级压缩：在LLM的推理阶段，进一步进行视频级压缩，依据文本令牌与视觉令牌之间的相关性，逐步减少不必要的视觉信息，以提高计算效率。
多阶段短到长学习：模型通过多阶段的学习策略，首先对短视频进行训练，然后逐步引入长视频，确保模型在不同时间尺度下都能有效理解内容。
在实验部分，VideoChat-Flash经过多个基准测试的验证，显示出其在长视频理解任务中的卓越表现。通过与现有的主流模型进行比较，VideoChat-Flash在多个标准数据集上达到了领先的性能，尤其是在“针在视频干草堆”任务中，模型在处理10,000帧的情况下取得了99.1%的准确率。此外，模型的计算负载显著低于其他同类模型，证明了其在高效性和效果上的双重优势。实验还表明，通过层次化压缩和动态采样策略，VideoChat-Flash在处理长视频时不仅提高了理解能力，同时也优化了计算资源的使用。
在VideoChat-Flash的开发中，研究者们想出了一个聪明的方法来处理超长视频。首先，他们将视频分成小段，像把一部电影剪成几集一样。这样，每一小段视频就可以更容易地被理解。接下来，他们使用一种特殊的技术来压缩这些小段中的视觉信息，保留重要的细节，同时减少需要处理的数据量。就像在写文章时，只保留最重要的句子。最后，模型通过不同阶段的学习，从短视频开始，再逐渐过渡到长视频，确保它在各种情况下都能理解内容。这种方法不仅让模型在理解长视频时表现得更好，还让它在计算时更加高效，节省了很多资源。
2025-01-02｜HKU,ShanghaiAILab
http ://arxiv.org/abs/2501.01428v1https ://gpt4scene.github.io/
在近年来，尽管2D视觉-语言模型（VLMs）在图像与文本理解任务上取得了显著进展，但在3D空间理解方面仍显不足。为了解决这一问题，研究者们提出了GPT4Scene，这是一种新颖的视觉提示范式，旨在通过视频输入来增强VLMs对3D场景的理解。GPT4Scene通过构建鸟瞰图（BEV）来提供全局场景信息，结合空间-时间对象标记（STOmarkers），从而实现视频帧与BEV图像之间的对应关系。这种方法不仅提升了VLMs在3D问答、密集标注和视觉定位任务中的表现，还为小型开源VLMs提供了强有力的训练基础，显著提高了它们在3D理解任务上的能力。
GPT4Scene的框架分为几个关键步骤：
视频输入与3D重建：首先，从输入的视频中提取N帧图像，并利用已知的相机外参进行3D重建，生成相应的点云。
生成鸟瞰图（BEV）：基于重建的3D点云，渲染出BEV图像，这为VLMs提供了全局场景信息。
空间-时间对象标记：在BEV图像和视频帧中添加STO标记，以确保对象在空间和时间上的一致性。这些标记帮助VLMs在处理任务时聚焦于特定对象。
零-shot提示与微调：通过使用零-shot提示，GPT4Scene能够激活强大的闭源VLMs（如GPT-4o）进行3D场景理解。同时，对于小型开源VLMs（如Qwen2-VL），通过构建ScanAlign数据集并进行微调，显著提升了它们的3D理解能力。
在实验部分，研究者们对GPT4Scene的性能进行了全面评估，包括3D问答、密集标注和视觉定位等任务。实验结果表明，经过微调的Qwen2-VL在各项任务上均取得了显著的性能提升，尤其是在3D问答任务中，其BLEU-1分数提高了56.1%。此外，GPT4Scene在零-shot设置下的表现也优于现有的3D点云模型，展示了其在处理复杂3D场景时的潜力。通过消融实验，研究者们验证了BEV图像和STO标记在训练过程中的重要性，进一步证明了GPT4Scene框架的有效性。
在方法部分，我们可以将GPT4Scene的工作原理简单地理解为一个“视频到3D世界”的转换过程。首先，系统从一个场景视频中提取多帧图像，然后利用这些图像创建一个三维模型，就像制作一个虚拟的房间一样。接着，它将这个三维模型生成一个俯视图（BEV），就像从高空俯瞰整个房间的样子。为了帮助计算机更好地理解这些图像，研究者们在BEV图和视频帧中加上了标记，这些标记就像是在房间里为每个物体贴上标签，确保计算机在处理时知道每个物体的位置和身份。最后，通过一些特别的提示，GPT4Scene能够让强大的计算机模型（比如GPT-4o）理解这些3D场景，甚至在没有额外训练的情况下也能回答关于场景的问题。这样，GPT4Scene就能够帮助计算机更好地理解和互动于我们的三维世界。
2025-01-02｜UAdelaide,MicrosoftResearch,CUHK(SZ),MBZUAI
http ://arxiv.org/abs/2501.01163v1
3D-LLaVA是一种新型的3D大型多模态模型（3DLMM），旨在提升3D场景理解和人机交互的能力。当前的3DLMM在对话和推理方面展现出巨大潜力，但仍面临如何实现细粒度场景理解的挑战。3D-LLaVA通过采用简约的设计，整合了多种功能，能够直接处理点云数据，避免了复杂的处理流程。其核心是OmniSuperpointTransformer（OST），该模块集成了视觉特征选择、视觉提示编码和分割掩码解码三种功能，使得模型在多个3D任务中表现出色。通过统一的指令调优，3D-LLaVA在各类基准测试中取得了令人瞩目的成绩，展示了其作为智能助理在理解和与3D世界互动中的潜力。
3D-LLaVA的整体架构采用了OmniSuperpointTransformer（OST），实现了多个功能的整合。具体方法包括：
视觉特征选择：OST能够选择性保留重要的视觉特征，突出3D场景中的关键信息，同时降低计算开销。
视觉提示编码：模型通过将用户的视觉提示（如点击点或框选区域）映射到与视觉特征相同的嵌入空间，允许与3D场景的有效交互。
掩码解码：OST直接生成3D掩码，消除了对额外分割模块的依赖，使得模型更加简洁并具备自我包含性。
在训练阶段，OST与3D场景编码器共同进行预训练，采用混合监督策略，通过实例分割和2D到3D知识蒸馏来提升模型性能。最终，3D-LLaVA通过整合视觉和语言信息，形成统一的输入，能够有效地进行3D理解和任务执行。
在实验部分，3D-LLaVA在多个流行的3D视觉和语言理解数据集上进行了评估，包括ScanRefer、ScanQA和SQA3D等。通过与现有方法的比较，3D-LLaVA在多项任务上表现出色，特别是在3D问答和引用分割任务中，取得了最高的评估指标。具体来说，模型在ScanQA数据集上实现了92.6%的CiDEr得分，显著优于其他竞争对手。此外，模型的设计还允许其在不同任务之间进行灵活切换，展示了其作为通用3DLMM的潜力。实验结果表明，3D-LLaVA不仅提升了3D场景理解的准确性，还增强了与用户的互动能力。
3D-LLaVA的工作原理可以简单理解为一个智能助手，它能够理解和处理三维世界的信息。首先，它通过一种叫做OmniSuperpointTransformer的特殊工具来选择和处理重要的视觉信息，就像在一幅复杂的画中找到最重要的部分。然后，当用户给出指令或提示时，3D-LLaVA能够将这些视觉提示转化为模型可以理解的形式，从而准确地进行任务，比如在三维场景中找到特定的物体。最后，模型能够根据用户的描述生成相应的三维掩码，这就像给出一个区域的轮廓，帮助用户清晰地看到他们所描述的物体。通过这种方法，3D-LLaVA不仅能理解用户的语言，还能在三维空间中做出准确的反应。
