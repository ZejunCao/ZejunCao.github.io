2025-03-12｜ShanghaiAcademyofArtificialIntelligenceforScience,FDU｜⭐️🟡
http ://arxiv. org/abs/2503. 09279v1https ://sais-fuxi. github.io/projects/cockatiel/
本研究提出了一种新的视频详细描述生成框架，名为Cockatiel，旨在解决当前视频详细描述模型在细粒度对齐和人类偏好方面的不足。通过对现有模型的全面基准测试，识别出两大关键挑战：对特定描述方面的偏向性和与人类偏好的不一致性。为此，Cockatiel采用了三阶段训练管道，结合合成数据和人类偏好的训练，以提升视频描述生成的效果。Cockatiel-13B模型在VDCSCORE评测上取得了新的最优性能，并在与人类偏好的对齐方面超越了现有的其他模型，展示了其在生成细致视频描述方面的强大能力。
Cockatiel的训练方法分为三个阶段：第一阶段是人类偏好评分和数据选择，第二阶段是合成训练，第三阶段是模型蒸馏。具体步骤如下：
人类偏好评分：构建一个结构化的人类偏好评分数据集，通过对视频-描述对进行详细标注，生成五个维度的评分，包括对象、背景、相机运动等。
合成训练：使用经过评分的高质量描述作为训练数据，Cockatiel-13B模型在此基础上进行微调，以确保生成的描述在各个维度上与人类偏好高度一致。
模型蒸馏：为便于用户使用，进一步从Cockatiel-13B蒸馏出Cockatiel-8B模型，确保在保持性能的同时，降低模型复杂度。
在实验部分，研究首先通过定量评估对比Cockatiel-13B与其他现有模型的表现，结果显示Cockatiel在VDCSCORE评测中取得了显著的性能提升，特别是在细粒度描述的对齐方面。同时，进行了一系列的消融实验，分析不同选择策略和训练数据集规模对模型性能的影响。结果表明，采用基于评分的选择策略和适当的训练数据集规模能显著提高模型的描述质量。此外，进行的人类评估显示Cockatiel-13B在用户偏好上也明显优于其他基线模型，验证了其在实际应用中的有效性和可接受性。
Cockatiel的训练方法可以简单理解为一个三步走的过程。第一步，我们收集了大量人类对视频描述的评分，确保这些描述准确地反映视频的内容。第二步，我们利用这些高分描述来训练Cockatiel-13B模型，让它学习如何生成更好的视频描述。最后一步，我们从这个大型模型中提炼出一个更小、更易用的模型Cockatiel-8B，这样普通用户也能方便地使用。整个过程不仅让模型的描述更加细致、准确，还能让它更符合人们的阅读习惯和偏好。
2025-03-12｜ShanghaiTech,XDU,TeleAI
http ://arxiv. org/abs/2503. 09387v1
本论文提出了VideoScan，一个高效的视觉语言模型（VLM）推理框架，旨在实现实时视频交互，能够有效理解和保留流式视频输入，同时提供快速且准确的响应。视频理解面临的一个长期挑战是，由于视觉标记的数量庞大，计算开销显著增加。为了解决这个问题，VideoScan引入了一个单一的语义载体标记来表示每一帧，从而在推理过程中逐步减少计算和内存开销。该模型通过优化的帧级视觉特征聚合，确保生成的语义载体标记具有紧凑但语义丰富的表示。通过训练相应的关键值对，VideoScan能够保留来自前帧的上下文语义，从而实现高效的内存管理而不牺牲时间一致性。实验表明，VideoScan在多个离线和在线基准测试中表现优异，显著提升了推理速度和性能。
VideoScan的方法主要分为两个阶段：预填充和解码。首先，在预填充阶段，每帧视频通过一个变换器基础的视觉编码器进行编码，生成视觉嵌入。然后，通过对这些嵌入进行平均池化，构建出一个语义载体标记，该标记包含当前帧的全局语义信息。接着，该语义载体标记被存储在一个动态内存库中，以便在解码阶段使用。在解码时，所有需要的视觉信息均来自于这些语义载体标记，避免了重复计算，从而减少了内存消耗。此外，VideoScan还引入了一种记忆机制，用于动态保留显著的历史上下文，通过选择性保留来确保GPU内存的稳定性。整个推理过程旨在实现高效的流式视频理解，优化了计算延迟和内存使用。
在实验部分，研究者对VideoScan进行了全面评估，比较其在离线视频理解基准和在线流视频问答基准上的表现。实验结果显示，VideoScan在多个视频理解基准上均表现出色，使用单一语义载体标记处理每帧，并结合固定容量的内存机制，成功保留了长时间推理所需的关键标记。尤其是在实时视频交互场景中，VideoScan展示了其出色的推理速度和较低的延迟，显著优于现有的流媒体理解模型。通过对不同组件的消融研究，验证了语义载体设计和记忆机制对提升视频理解能力的重要性。这些结果表明VideoScan在实时应用中的潜力，尤其是在机器人、监控和互动系统等领域。
VideoScan的方法可以简单理解为一个两步走的过程。首先，在处理视频时，VideoScan会将每一帧的图像信息提取出来，并通过一个特殊的算法进行汇总，形成一个代表该帧的“语义载体”。这个载体就像是该帧的缩影，包含了重要的信息。接下来，在回答问题或进行互动时，VideoScan只需使用这些载体，而不需要再次处理每一帧的图像，从而大大加快了处理速度，减少了计算量。同时，VideoScan还有一个智能的记忆系统，可以根据需要保留重要的历史信息，以便在后续处理中使用。这种方法让VideoScan能够在实时场景中快速反应，同时保持高效的内存使用，确保系统的流畅性。
2025-03-12｜BUPT,GML(SZ),CSU
http ://arxiv. org/abs/2503. 09158v2
FaVChat是一种新型的多模态大语言模型（VMLLM），专门设计用于细粒度的面部视频理解。随着视频作为信息获取的主要媒介，其在理解人类面部特征方面的重要性日益凸显。然而，现有的VMLLM在细粒度面部理解方面的能力尚未得到充分开发。为了解决这一问题，FaVChat通过构建一个包含超过60,000个视频的庞大数据集，标注了83种细粒度面部属性，以丰富生成的文本描述。此外，FaVChat采用了一种混合模型架构，结合了通用视觉编码器和专用面部编码器，以提高面部特征的提取能力。通过采用渐进式训练策略，FaVChat逐步提高任务复杂性，从视频摘要到高质量视频问答，增强了模型对细粒度视觉细节的理解能力。
FaVChat的方法主要包括以下几个关键步骤：
数据集构建：收集并注释来自CelebV-HQ、FaceForensics++、HMDB51和YouTubeFaces的数据，确保多样的面部表情和动作。数据集包含35,666个高质量视频片段，涵盖83种面部属性。
多模态学习架构：采用双编码器结构，包括通用视觉编码器和专用面部编码器，后者能够提取更为细致的面部特征。同时，结合了MixtureofExperts（MoE）适配器以实现多源视觉特征的自适应融合。
渐进训练策略：分为三个阶段，首先进行视频摘要训练，随后转向中等难度的视频问答，最后对高质量的细粒度问答对进行微调。这种策略旨在逐步提升模型对面部细节的理解能力。
零样本评估：在多个公共基准上进行广泛的零样本评估，验证FaVChat在多项任务中的优越性能。
在实验中，FaVChat被评估在多个任务上的表现，包括面部表情分析、情感识别和可解释的情感推理。通过与现有的高性能VMLLM进行比较，FaVChat在细粒度面部视频理解任务中展现出卓越的能力。例如，在情感分类任务中，FaVChat在DFEW数据集上显著超越了其他模型，显示出其在细微面部动态捕捉方面的优势。此外，进行了一系列消融实验，以分析各个组件对模型性能的贡献。结果表明，专用面部编码器和多粒度学习策略对提升细粒度面部理解至关重要。最终，FaVChat在多个评估指标上均表现出色，证明了其在细粒度视频分析领域的有效性。
FaVChat的工作原理可以简单理解为一个“超级视频分析师”。首先，它从各种来源收集了大量视频，这些视频展示了不同人的面部特征和表情。接下来，FaVChat利用两种不同的工具来分析视频：一种是用来处理所有视频内容的通用工具，另一种则专门聚焦于面部特征的细节。通过这种双重分析，FaVChat能够更好地理解视频中的人脸细节，比如皮肤颜色、发型和面部表情。为了让模型更聪明，研究人员设计了一种逐步训练的方法，先让它学习简单的内容，然后逐渐增加难度。这样，FaVChat就能在面对复杂问题时，提供更准确的答案，比如识别情感或分析面部表情。这种方法使得FaVChat在理解视频内容时，比其他模型更加出色。
2025-03-12｜RUC,BAAI,CAS,UTrento,RUC
http ://arxiv. org/abs/2503. 09149v1
长视频理解（LVU）是一个在视频分析、自动驾驶及交互式人工智能等现实应用中至关重要的任务。然而，传统的多模态大语言模型（MLLMs）在处理长视频时面临显著挑战，主要由于其对短视觉输入的设计限制。为了解决这一问题，本文提出了一种名为MemVid的新型检索增强生成（RAG）框架。该方法受到人类认知记忆的启发，旨在通过更有效地处理长视频中的信息来提升理解能力。MemVid通过四个基本步骤来执行任务：首先对视频信息进行整体记忆；其次，基于记忆推理任务所需的信息；接着，检索与信息需求相关的关键时刻；最后，聚焦于检索到的时刻以生成最终答案。通过这种方法，MemVid显著提高了长视频理解的性能，并在多个基准测试中超越了现有的RAG方法和流行的LVU模型。
MemVid的工作流程可以分为四个主要步骤：
记忆构建：将整个视频编码为结构化的记忆，以捕捉长程依赖关系和上下文信息。
推理阶段：利用记忆模型推断出与特定问题相关的潜在信息需求，通过分析问题来生成检索线索。
检索过程：根据生成的检索线索，从视频中提取与之相关的关键时刻，以形成可供后续处理的候选集。
聚焦生成：结合检索到的证据时刻和原始问题，综合生成最终答案。为了提高系统的推理能力，MemVid还采用了一种课程学习策略，先通过监督学习建立基础，然后通过强化学习逐步优化推理过程。这种方法确保了推理能力的逐步提升，从而增强了整体性能。
为验证MemVid的有效性，研究团队在多个流行的长视频理解基准上进行了广泛的评估，包括MLVU、VideoMME和LVBench。实验结果显示，MemVid在各项任务中均显著优于现有的RAG方法和其他流行的长视频模型，证明了其在处理复杂和上下文依赖信息方面的优势。具体而言，MemVid在关键事件检索、事件理解和推理能力方面表现出色，尤其是在长视频的复杂性处理上，显示出其在长视频理解领域的创新性和有效性。此外，研究还展示了MemVid在不同模型和输入条件下的适应性，进一步证明了其广泛的应用潜力。
MemVid的工作原理可以简单理解为四个步骤。首先，它会把整个视频的信息保存下来，就像人类记忆一样，形成一个全面的记忆库。接着，当有问题出现时，MemVid会根据这个记忆库来推理出需要的信息，比如需要关注哪些内容。然后，它会根据这些推理出的信息，从视频中找到相关的关键时刻，像是在寻找答案的线索。最后，MemVid会把这些找到的线索和问题结合起来，生成一个完整的答案。为了让这个过程更有效，MemVid还会先进行一些基础训练，确保在遇到更复杂的问题时能够更好地找到答案。通过这样的方式，MemVid能够更准确地理解和回答关于长视频的问题。
2025-03-12｜PKU,SalesforceResearch
http ://arxiv. org/abs/2503. 09146v1https ://generative-sampler. github.io
本研究提出了一种新的生成框架采样器（GenerativeFrameSampler,GenS），旨在提升长视频理解的效率。尽管视频大型语言模型（VideoLLMs）在处理视频内容方面取得了显著进展，但对长视频（例如包含数千帧的视频）的有效理解仍然面临巨大的计算挑战。传统的采样方法如均匀采样和每秒帧数（FPS）采样在处理长视频时，往往导致信息丢失和高内存消耗。为了应对这一问题，GenS通过集成到轻量级的VideoLLM中，利用其视觉-语言能力来识别与用户问题相关的帧，从而提升长视频的感知能力。研究表明，GenS在多个视频理解基准上显著提高了性能，尤其是在长视频问答任务中。
GenS的设计围绕着两个主要挑战展开：第一，缺乏足够的训练数据来支持有效的帧采样；第二，现有的数据集往往缺乏对相关帧的密集标注。为了解决这些问题，研究者构建了一个名为GenS-Video-150K的大规模视频指令数据集，包含对相关帧的密集标注和细粒度置信度评分。GenS的方法包括四个主要阶段：1)密集视频帧字幕生成，2)基于生成的字幕构建视频问答对，3)扩展相关帧集合以提高检索能力，4)对候选帧进行细粒度置信度评分。通过这些步骤，GenS能够高效识别与用户指令相关的帧，并通过动态框架索引机制与现有的视频问答助手无缝集成，提升视频理解的整体性能。
在实验部分，GenS被评估在多个长视频基准上，包括LongVideoBench、MLVU和HourVideo等。结果显示，GenS显著提升了各种视频问答模型的性能，尤其是在处理长视频时。通过与传统的均匀采样方法进行比较，GenS在多个模型上实现了显著的准确性提升。例如，在LongVideoBench上，集成GenS的LLaVA-Video-72B模型达到了66. 8%的准确率，较之前提高了4. 3个百分点。研究还对GenS的有效性进行了细致分析，结果表明其在复杂的语言理解和时间关系建模方面表现优异。此外，GenS在处理长视频时的动态帧选择能力，使其在多个问答任务中取得了领先的表现，显示出其在长视频理解领域的潜力。
在本研究中，研究人员提出了一种新的方法，叫做生成框架采样器（GenS），来帮助计算机更好地理解长视频。传统的方法在处理长视频时，往往会遗漏一些重要的信息，导致理解不准确。GenS的工作原理是先从视频中提取出重要的帧，然后根据用户提出的问题，选择与之相关的帧。这个过程分为几个步骤：首先，生成视频中每一帧的描述；接着，利用这些描述来创建问题和答案对；然后，扩展相关帧的集合，以便更全面地覆盖视频内容；最后，对这些帧进行评分，判断它们与问题的相关性。通过这些步骤，GenS能够有效地帮助视频问答助手更好地理解长视频，提高回答的准确性。
2025-03-12｜UNCChapelHill,MetaAI｜CVPR2025
http ://arxiv. org/abs/2503. 09590v2https ://sites. google.com/view/bimba-mllm
在视频问答（VQA）领域，处理长视频时的关键挑战在于有效提取相关信息和建模长距离依赖关系。传统的自注意力机制虽然在序列建模上表现优越，但在处理长视频时，其计算成本呈现二次增长，这使得处理大量时空令牌变得不切实际。现有方法通常采用压缩策略来降低计算负担，但这些方法往往忽略了重要的时空信息。为此，本文提出了BIMBA（BidirectionalInterleavedMambaforBetterAnswers），一种高效的状态空间模型，旨在将长视频压缩为信息丰富的令牌序列，以便进行高效的语言模型处理。通过选择性扫描算法，BIMBA能够有效提取高维视频中的关键信息，并在多个长视频VQA基准测试中实现了最先进的准确率。
BIMBA的核心架构包括三个主要组件：图像编码器、时空令牌选择器和大型语言模型（LLM）解码器。首先，图像编码器独立处理每一帧视频，提取出空间特征（令牌）。接着，时空令牌选择器的主要任务是捕捉长距离依赖关系，并从冗余信息中选择最相关的令牌。为此，它利用选择性扫描机制，能够以线性计算成本高效建模时空依赖。该选择器还采用了双向扫描策略，增强了模型对时空结构的捕捉能力。最后，经过选择和压缩的令牌序列与文本问题一起输入到LLM解码器中，生成相应的文本回答。BIMBA的设计通过交错查询和双向选择性扫描等创新，显著提高了模型在处理长视频时的表现。
在实验部分，BIMBA在多个长视频问答基准（如PerceptionTest、NExT-QA、EgoSchema等）上进行了评估。研究团队对比了BIMBA与其他压缩方法的表现，发现BIMBA在准确率和计算效率上均优于基线模型。实验结果显示，BIMBA在处理不同数量的输入令牌时，准确率始终保持在领先地位，尤其在长视频的情况下，模型能够有效提取关键信息而不丢失重要的时空关系。此外，实验还分析了不同设计选择对模型性能的影响，验证了双向选择性扫描和交错查询的有效性。这些结果表明，BIMBA不仅在长视频理解上表现卓越，还具备良好的计算效率。
BIMBA的工作原理可以简单理解为一个高效的信息筛选和处理系统。首先，它会从每一帧视频中提取出很多特征，就像从一堆照片中找出最重要的部分。然后，BIMBA会使用一种聪明的策略，选择出对回答问题最有帮助的特征，而不是把所有的特征都放在一起，这样可以避免信息过多导致的混乱。这个过程就像在阅读一本长书时，只挑选出关键段落来理解故事情节。最后，BIMBA把这些精选的特征和问题一起送入一个智能的语言模型，生成准确的回答。通过这种方式，BIMBA能够在处理长视频时既保持高效又确保信息的完整性。
2025-03-12｜HIT(SZ),PengchengLab,SDJZU,SDU,Kuaishou
http ://arxiv. org/abs/2503. 09143v1https ://egovisiongroup. github.io/Exo2Ego. github.io/
本文提出了一种新的多模态大语言模型（MLLM），名为Exo2Ego，旨在提升对第一人称（自我中心）视频的理解能力。现有的MLLM主要聚焦于第三人称（外部视角）视频，忽视了自我中心视频的独特特性。为了克服这一局限，研究者们设计了一种学习外部和自我中心领域之间映射的框架，利用现有模型中的外部知识来增强自我中心视频的理解。此外，数据获取成本高导致数据规模受限，进一步影响了模型性能。为了解决这些挑战，研究团队构建了Ego-ExoClip数据集，包含110万个同步的自我中心和外部视频-文本对，并提出了一个包含三个阶段的渐进训练流程，以推动模型的性能提升。
Exo2Ego模型的训练方法分为三个主要阶段：教师自我准备、教师-学生指导和学生自我实践。具体步骤如下：
教师自我准备：冻结外部视觉编码器的参数，使用Ego-ExoClip数据集对其进行微调，以确保其能够有效理解外部视频。
教师-学生指导：在这一阶段，冻结外部视觉编码器的参数，并微调自我中心视觉编码器。通过建立一个映射函数，将外部知识转移到自我中心领域，同时应用循环一致性损失（CCL）来保证映射的准确性。
学生自我实践：最后，利用EgoIT数据集增强模型的指令跟随能力。通过结合自我中心样本和映射的外部样本的表示，形成全面的语义输入，以提高模型在各种自我中心任务中的表现。
在实验部分，研究团队通过EgoBench基准测试对Exo2Ego模型进行了全面评估。实验结果显示，该模型在多个自我中心任务上显著优于现有的MLLM，如VideoChat2和VideoLLaMA2。在不同的数据集上，Exo2Ego在推理、记忆和规划等任务上均表现优异，验证了其在理解自我中心视频方面的有效性。通过对比不同模型架构和参数更新策略，研究者们还发现，映射函数和指令调优对模型性能起到了关键作用。此外，实验还强调了跨领域知识转移的重要性，证明了Exo2Ego在自我中心视频理解中的潜力。
Exo2Ego模型的训练方法可以简单理解为三个步骤。首先，教师自我准备阶段就像一个老师在自我学习，先把外部视频的知识掌握好。接着，在教师-学生指导阶段，老师会教学生（自我中心视频编码器）如何把这些知识运用到自我中心的视频中。最后，学生自我实践阶段，学生会根据老师的教导，结合自己学到的知识，进行更多的练习，以提高对自我中心视频的理解能力。这个过程就像是一个学习的循环，老师和学生之间不断互动，最终使得学生在面对自我中心视频时，能够更准确地理解和回答问题。
2025-03-12｜NUS｜CVPR2025
http ://arxiv. org/abs/2503. 09402v1https ://github. com/showlab/VLog
VLog是一种新颖的视频理解框架，旨在通过生成式检索叙述词汇来提升视频内容的解析能力。传统的视频语言模型通常依赖于子词词汇，而VLog则将视频叙述视为一种独特的词汇形式，能够更有效地捕捉日常活动的关键事件。该框架基于轻量级的GPT-2模型，结合了生成模型的推理能力与对比检索的高效性，提出了三项关键创新：生成式检索模型、分层词汇构建和词汇更新策略。这些创新使VLog能够在处理长视频时实现显著的速度提升，并在生成简洁且上下文相关的叙述方面表现出色。通过引入VidCap-Eval开发集，VLog的有效性在多个数据集上得到了验证，展现了其在视频理解领域的潜力。
VLog的核心方法包括三个主要部分：生成式检索架构、叙述对编码（NPE）和分层索引。首先，生成式检索架构结合了生成模型和检索模型的优势，利用检索令牌在生成叙述时考虑视觉输入和查询信息。其次，叙述对编码方法将每个叙述分解为前缀和后缀，使得词汇构建更加高效。该方法通过识别共享前缀的叙述，生成一个包含丰富上下文信息的词汇集。最后，VLog采用分层索引策略，通过先识别视频场景（如厨房）来快速检索相关的前缀叙述，然后再从后缀词汇中提取具体信息。这种方法不仅提高了检索速度，还增强了模型的推理能力，使其能够处理复杂的查询。
在实验部分，VLog的性能通过多个基准测试进行评估，包括EgoSchema、COIN和HiREST等数据集。实验旨在回答VLog的主要优势、适应性和词汇升级策略的有效性。结果显示，VLog在生成叙述的准确性和处理速度上均优于传统模型，尤其在复杂推理任务中表现突出。通过对比不同模型的性能，VLog在处理长视频时实现了高达20倍的速度提升。此外，词汇升级策略有效解决了新事件的识别问题，确保VLog能够适应不断变化的叙述需求。实验结果表明，VLog在视频理解任务中具备显著的实用性和灵活性。
VLog的工作原理可以简单理解为一个智能视频叙述助手。它通过识别视频中的场景和动作，生成简洁的描述。首先，VLog会将视频中的每个动作分成两部分：前缀和后缀。前缀是动作的开头部分，比如“切土豆”，而后缀则提供更多细节，比如“用左手切”。接下来，VLog会根据视频的场景（例如厨房）快速找到相关的前缀描述，并结合后缀信息生成完整的叙述。这样的设计让VLog不仅能快速生成描述，还能理解视频中的复杂关系，比如“接下来会发生什么”。总的来说，VLog通过聪明的词汇管理和快速检索，帮助我们更好地理解视频内容。
2025-03-12｜OsakaU,CyberAgent
http ://arxiv. org/abs/2503. 09027v1https ://github. com/pangzss/MeCo
在视频理解模型中，基于用户查询的事件定位至关重要。近年来的研究主要集中在利用视频大语言模型（VideoLLMs）生成事件边界时间戳，以处理时间定位任务。然而，这些方法往往未能充分利用LLMs强大的语义理解能力。为此，本文提出了一种新颖的无时间戳框架MeCo，旨在让视频LLMs充分发挥其内在的语义能力进行时间定位。MeCo通过将视频划分为整体事件和过渡段落，而不是输出时间戳来处理视频内容。通过引入结构性标记生成和基础模块，MeCo能够更好地理解视频的时间结构。此外，本文还提出了一种聚焦查询的字幕生成任务，以提取细粒度的事件特征，提升定位性能。实验结果表明，MeCo在多种时间定位任务中均优于传统的基于边界的方法，展示了语义驱动方法在视频时间定位中的优势。
MeCo的核心方法包括两个主要部分：结构性标记生成和结构性标记基础。首先，模型通过生成两种类型的标记——事件标记（）和过渡标记（），来划分视频为连续的事件和背景过渡段。这一过程利用了视频LLMs对时间结构的理解能力。其次，为了将结构性标记与相应的视频段落进行映射，MeCo引入了结构性标记基础模块，通过最大化结构性标记与对应视频段之间的似然性来优化模型。最后，MeCo还设计了一个聚焦查询的字幕生成任务，促使模型提取与查询事件相关的详细语义信息，从而提升时间定位的精度。这一方法不仅关注事件边界的生成，还强调了对事件细节的深入理解，以实现更准确的定位。
在实验部分，MeCo的性能在多个基准上进行了评估，包括E. T.Bench、Charades-STA和QVHighlights。实验结果显示，MeCo在零-shot条件下的表现显著优于现有的时间定位方法。具体而言，MeCo在E. T.Bench的多个任务中取得了最高的F1分数，尤其在时间视频定位和密集字幕生成任务中表现突出。此外，MeCo还展示了在复杂推理任务中的优势，证明其在处理多样化视频内容时的有效性。通过与传统的基于边界的方法进行对比，MeCo显示出其语义驱动的方法在时间定位中的潜力，进一步验证了其在实际应用中的可行性和优越性。
MeCo的方法可以简单理解为一个两步走的过程。首先，模型会把视频内容分成两个部分：事件和过渡。事件是视频中发生的具体动作或情节，比如“一个人正在切菜”，而过渡则是背景信息，比如“厨房的全景”。为了做到这一点，MeCo会生成两种特殊的标记，分别表示事件和过渡。这就像给视频加上标签，让模型知道每一部分的内容。第二步是生成与用户查询相关的详细字幕。例如，当用户询问某个特定事件时，模型会生成描述这个事件的详细信息。通过这两步，MeCo不仅能准确找到视频中的特定片段，还能理解和描述这些片段的具体内容，从而提升用户体验。
