http ://arxiv.org/abs/2502.02501v1https ://yufanchen96.github.io/projects/GraphDoc
在现代文档分析中，理解文档的结构布局是至关重要的。传统的文档布局分析（DLA）方法主要集中在检测和分类基本文档元素，如文本块、图像和表格，往往无法捕捉到不同组件之间复杂的空间和逻辑关系。这种局限性使得基于DLA的模型难以实现更深层次的理解。为了解决这一问题，本文提出了图基文档结构分析（gDSA）任务，旨在不仅检测文档元素，还生成以图结构形式表示的空间和逻辑关系，从而实现对文档的整体和直观理解。为此，我们构建了一个包含8万张文档图像及413万个关系注释的GraphDoc数据集，使模型能够执行诸如阅读顺序预测、层次结构分析和复杂相互关系推断等多项任务。
本研究的核心在于GraphDoc数据集的构建及其相关任务的定义。GraphDoc数据集包含两个主要任务：文档布局分析（DLA）和图基文档结构分析（gDSA）。对于DLA任务，我们利用带标签的边界框提取文档中的布局信息，并将其分类为11种不同的元素。gDSA任务则旨在提取布局元素之间的关系图，节点代表布局元素，边代表元素间的关系，如父子关系、顺序和引用。为支持gDSA任务，我们还定义了两个子任务：阅读顺序预测（ROP）和层次结构分析（HSA）。此外，我们提出了文档关系图生成器（DRGG），一个端到端架构，旨在从文档布局中生成关系图，通过结合对象检测和关系预测来捕捉文档元素之间的复杂关系。
为了评估所提出的DRGG框架在GraphDoc数据集上的有效性，我们与多种最先进的方法进行了比较，包括DETR、DeformableDETR、DINO和RoDLA等。实验结果表明，结合InternImage主干网络和RoDLA检测器的DRGG框架在DLA任务中达到了81.5%的mAP，优于其他组合。此外，在gDSA任务中，DRGG在多项评估指标上表现出色，mR@0.5达到了30.7%，mAP@0.5、0.75和0.95分别为57.6%、56.3%和46.5%。这些结果表明，DRGG在捕捉文档结构的复杂性方面具有显著优势，尤其是在处理空间和逻辑关系时。
在我们的方法部分，我们首先建立了一个新的数据集，称为GraphDoc，它帮助计算机更好地理解文档的布局和结构。这个数据集包含了很多文档图像，并为每个图像中的不同元素（例如标题、文本、图片等）标注了它们之间的关系。我们将这些关系分为两种类型：空间关系（比如一个元素在另一个元素的上面或下面）和逻辑关系（比如一个标题是某段文本的父级）。通过这种方式，计算机不仅可以识别文档中的各种元素，还可以理解它们是如何相互关联的。此外，我们还设计了一种新的模型，叫做DRGG，它能够从文档中提取这些关系，帮助计算机更好地理解文档的整体结构。这种方法使得计算机能够像人类一样，理解和分析复杂的文档布局。

http ://arxiv.org/abs/2502.01977v1https ://autogui-project.github.io/
随着视觉语言模型（VLMs）在用户界面（UI）理解领域的广泛应用，如何有效地标注UI元素的功能性描述成为了一个重要的研究课题。现有的UI数据集通常只提供大型无上下文的元素标注或小规模的上下文功能描述，无法满足大规模训练的需求。为此，本文提出了AutoGUI，一个自动化的UI数据标注管道，旨在通过大规模的功能性描述标注来提升VLMs的UI理解能力。该方法利用大型语言模型（LLMs）来推断UI元素的功能，通过模拟交互前后的UI内容变化来进行标注。此外，为了提高标注质量，AutoGUI引入了LLM辅助的拒绝和验证机制，减少了人工干预的需要。最终，构建了一个包含704,000个高质量功能标注的AutoGUI-704k数据集，显著提升了VLMs在UI定位任务中的表现。
AutoGUI的设计包括几个关键步骤，以确保高效且准确地生成UI元素的功能性标注：64.收集UI交互轨迹：首先，AutoGUI会通过爬虫技术收集用户与UI元素交互的轨迹，记录每次交互前后的UI状态变化。这些轨迹为后续功能性标注提供了基础数据。65.功能性标注：在收集到的交互轨迹上，使用LLM分析UI内容的变化。例如，当用户点击某个按钮后，若出现新的下拉菜单，LLM会推测该按钮的功能为“激活下拉菜单”。66.无效样本剔除：为了确保数据质量，AutoGUI引入了LLM辅助的拒绝机制，剔除如未完全加载的UI等无效样本。通过手动规则和LLM的结合，进一步提高了标注的准确性。67.标注验证：在标注完成后，AutoGUI还会通过双重LLM验证系统，检查标注的准确性，确保每个功能描述与实际UI元素的作用一致。68.数据集生成：最终，经过上述步骤处理的数据将被整理成高质量的功能性标注数据集，用于后续的模型训练和评估。
在实验部分，研究团队使用AutoGUI-704k数据集对多种视觉语言模型进行了微调，评估其在UI定位任务上的表现。实验结果表明，经过AutoGUI数据集微调的模型在多个基准测试上均表现出显著的性能提升，验证了AutoGUI数据集在增强VLMsUI理解能力方面的有效性。此外，研究还探讨了数据集规模对模型性能的影响，发现随着数据集规模的增加，模型的UI定位准确率也显著提高。这些结果不仅展示了AutoGUI在数据生成上的优势，也为未来的UI理解研究提供了新的思路。
AutoGUI的工作原理可以简单理解为一个智能的标注助手。首先，它会观察用户如何与应用程序的界面互动，比如点击按钮或输入信息。接着，它会记录这些互动前后的变化，比如一个按钮被点击后是否出现了新的选项。通过这些变化，AutoGUI会利用强大的语言模型来猜测每个按钮或输入框的功能是什么，比如“这个按钮可以打开一个下拉菜单”。为了确保这些猜测是准确的，AutoGUI还会检查这些标注是否合理，甚至会通过两个不同的智能助手进行交叉验证。最终，经过这些步骤，AutoGUI生成了一个包含大量高质量功能描述的数据集，帮助计算机更好地理解和使用用户界面。
2025-02-04｜USTC,NTU,CASIA,NUS
http ://arxiv.org/abs/2502.01943v2https ://github.com/injadlu/DAMA
在多模态大型语言模型（MLLMs）的研究中，直接偏好优化（DPO）已被证明在与人类偏好对齐方面的有效性。然而，现有方法在处理不同难度的数据时表现出不平衡的响应性，容易对易于区分的数据过拟合，而对难以区分的数据则表现出欠拟合。为了解决这一问题，本文提出了一种新的方法——数据和模型感知的直接偏好优化（DAMO）。该方法通过两个关键策略动态调整优化过程：一是数据感知策略，考虑数据的难度；二是模型感知策略，实时整合模型的响应。通过这两种策略的结合，DAMO显著提高了模型的可信度和在各种任务上的有效性，并在多个基准测试中超越了现有的最先进技术。
DAMO方法的核心在于其两种创新机制：数据感知偏好优化和模型感知偏好优化。具体而言：69.数据感知偏好优化：通过使用CLIP模型计算图像和文本之间的相似性分数，量化数据的难度。该过程首先将偏好响应分解为子句，便于细粒度估计。接着，计算各子句与图像的相似度，并根据相似度差异来评估数据的难度。通过动态调整超参数β，强化对“易于区分”样本的正则化，同时对“难以区分”的样本放宽约束，从而防止过拟合和欠拟合。
模型感知偏好优化：通过计算偏好响应与拒绝响应之间的隐含奖励差距，来估计模型的当前响应性。为了提高估计的稳健性，该过程对奖励差距进行归一化，并去除异常值。将这一估计整合进优化过程，动态调整β，使模型能够根据当前的响应性进行有效优化。
通过这两种策略的结合，DAMO实现了对数据难度和模型响应性的实时适应，从而在多项评估基准中展示了强大的对齐性能。
在实验部分，研究者们评估了DAMO在五个基准测试上的有效性，包括三个针对幻觉的基准和两个通用基准。实验结果显示，DAMO在各项任务中均显著优于现有的最先进方法。具体而言，在ObjectHalBench基准上，DAMO-7B模型的响应级和提及级幻觉率分别降低了90.9%和95.3%。此外，DAMO-13B在AMBER基准上获得了84.3%的准确率和88.1的F1分数，超越了GPT-4V，展现出其在处理多模态数据时的卓越能力。实验还通过消融研究分析了不同组件对模型性能的影响，结果表明，数据和模型感知策略的结合显著提升了整体鲁棒性。
在DAMO方法中，我们主要使用了两种新颖的策略来帮助模型更好地理解和处理数据。首先，数据感知策略就像给模型配了一双“眼镜”，让它能更清晰地看到哪些数据是容易理解的，哪些是比较困难的。我们通过比较图像和文本之间的相似性，来判断数据的难度，并根据这个难度来调整模型的学习方式。其次，模型感知策略则是让模型能够“自我反省”，它会根据自己在处理不同数据时的表现来判断自己是否需要调整学习策略。这样一来，模型就能更灵活地应对各种情况，避免在简单任务上过于自信而在复杂任务上失误。总的来说，这种方法让模型在理解和生成内容时变得更加可靠和准确。

本研究提出了COCONut-PanCap数据集，旨在增强全景分割与基于图像的描述生成能力。该数据集在COCO数据集的基础上，结合了先进的COCONut全景掩模，解决了现有图像-文本数据集中缺乏详细场景描述的限制。COCONut-PanCap包含精细的区域级描述，能够为图像理解与生成模型提供丰富的训练数据。实验结果表明，该数据集在理解和生成任务上显著提升了模型性能，建立了新的基准，为多模态学习提供了重要的支持。
本研究的方法主要包括以下几个步骤：
数据集构建：利用COCO数据集，结合人类标注的全景分割掩模，生成包含143K图像的详细描述数据集。
标注流程：采用一个高效的标注流程，通过人类编辑的草稿与全景掩模相结合，确保生成的描述既全面又准确。
多任务学习：设计了一个结合全景分割与基于图像的描述生成的新任务（PGC），并为此建立了评估指标与基准。
模型训练：利用生成的高质量描述进行视觉语言模型（VLM）的训练，优化模型在细粒度图像理解与生成任务中的表现。
实验部分评估了COCONut-PanCap数据集在多个下游任务上的有效性，包括详细描述生成、全景分割及文本到图像生成。研究通过与多种基线模型的对比，展示了所提方法在各项指标上的优势。具体实验结果表明，使用COCONut-PanCap数据集训练的模型在详细描述生成和全景分割任务中均取得了显著的性能提升，特别是在准确性和生成质量方面。通过对不同模型的评估，验证了该数据集在多模态理解和生成任务中的广泛适用性。
在这项研究中，我们创建了一个新的数据集，叫做COCONut-PanCap，目的是帮助计算机更好地理解和描述图像。我们的工作分为几个简单的步骤。首先，我们从一个叫COCO的图像库中提取了大量的图片，并为每张图片添加了详细的描述和分割信息。接下来，我们设计了一种高效的标注方法，确保每个描述都准确无误，能够清晰地反映出图像中的每个元素。然后，我们结合全景分割和描述生成两个任务，让计算机在学习时可以同时处理这两方面的信息。最后，我们通过一系列实验，证明了这个新数据集在帮助计算机理解图片和生成描述方面的有效性。通过这些方法，我们希望能推动计算机视觉领域的发展，使得计算机在处理图像时更聪明、更准确。
2025-02-04｜NKU,UAB,MBZUAI,LinköpingU,SoftBank｜ICLR2025
http ://arxiv.org/abs/2502.02215v1https ://sen-mao.github.io/InterLCM-Page/
本文提出了一种新颖的盲面部修复（BFR）框架，称为InterLCM，旨在从低质量（LQ）图像中恢复高质量（HQ）图像。传统的扩散模型在处理此类任务时存在一些关键局限性，包括语义一致性不足和对多次去噪迭代的依赖。为了解决这些问题，InterLCM利用潜在一致性模型（LCM），通过将低质量图像视为LCM的中间状态，从而实现更高的语义一致性和恢复质量。该方法不仅在合成数据集上表现出色，还在真实场景中展示了良好的恢复效果，尤其是在处理复杂的图像退化时。
InterLCM的核心在于其创新的框架设计，主要包括以下几个部分：5.潜在一致性模型（LCM）：该模型通过学习噪声到数据的映射，保持了更好的语义一致性。6.视觉模块（VisualModule）：该模块负责从低质量图像中提取视觉特征，提供面部特定的语义信息，以替代传统的文本提示。7.空间编码器（SpatialEncoder）：该编码器用于捕捉和增强图像的结构信息，确保恢复图像的整体结构保持一致。8.训练目标：通过结合重建损失、感知损失和对抗损失，InterLCM能够有效地优化模型，提升恢复图像的质量。
InterLCM通过将低质量图像视为LCM处理中的中间状态，利用LCM的噪声添加过程进行恢复，从而实现更高效的图像修复。
在实验部分，我们在多个合成和真实世界的数据集上评估了InterLCM的表现。通过与现有的多种基线方法进行对比，InterLCM在图像质量指标（如LPIPS、FID和MUSIQ）上均取得了优异的成绩。结果表明，InterLCM不仅在图像恢复的定量评估中表现突出，在视觉效果上也优于其他方法，尤其是在处理真实场景中的复杂退化时。此外，实验还验证了视觉模块和空间编码器对恢复质量的贡献，表明这两者的结合能够有效提升图像的细节和结构一致性。
InterLCM方法的核心在于它如何处理低质量图像。首先，它将低质量图像视为一个“中间状态”，这意味着它在恢复高质量图像时，实际上是从一个更接近真实的图像开始。为了让模型更好地理解这些图像，InterLCM引入了两个重要的组件：一个是视觉模块，它能够从低质量图像中提取出有关面部的特征；另一个是空间编码器，它帮助模型保持图像的整体结构。通过这种方式，InterLCM不仅能恢复出更清晰的面部细节，还能确保这些细节与原始图像的内容一致。总的来说，InterLCM通过聪明地利用这些技术，显著提升了图像恢复的效果。
2025-02-04｜SJTU,Harvard,SCUT,HuaweiNoah’sArkLab
http ://arxiv.org/abs/2502.01993v2https ://github.com/JianzeLi-114/FluxSR
FluxSR是一种新颖的一步扩散图像超分辨率（Real-ISR）技术，旨在提高从低质量图像恢复高质量图像的能力。该方法基于FLUX的流匹配模型，通过引入FlowTrajectoryDistillation（FTD）来将多步流匹配模型提炼为一步的超分辨率模型。与传统的多步扩散模型相比，FluxSR显著降低了计算成本和推理延迟，并在图像质量上取得了更好的效果。通过引入TV-LPIPS作为感知损失，并采用AttentionDiversificationLoss（ADL）来减少生成图像中的高频伪影，FluxSR在生成真实感图像方面表现出色。实验表明，该方法在多个基准测试中超越了现有的一步扩散图像超分辨率方法。
FluxSR的核心方法包括三个主要组成部分：9.**FlowTrajectoryDistillation(FTD)**：该方法通过保持原始的文本到图像（T2I）流不变，学习低分辨率（LR）到高分辨率（HR）流的生成轨迹。这一过程确保了生成图像的真实感和细节保留。10.大模型友好的训练策略：为了提高训练效率，FluxSR不依赖于额外的教师模型，而是利用教师模型在离线模式下生成的大量流数据。这减少了内存消耗和计算开销，使得训练过程更加高效。11.损失函数的改进：引入TV-LPIPS作为感知损失，强调高频细节的恢复，并结合ADL作为正则化项，提升了注意力模块中不同token的多样性。这些改进有效地减少了生成图像中的伪影。
在实验部分，FluxSR在多个数据集上进行了评估，包括DIV2K-val和RealSR等。通过生成2400个1024x1024的噪声-图像对进行训练，FluxSR在无参考和有参考的评估指标上都表现出色。实验结果表明，FluxSR在图像质量、视觉真实感和细节恢复上优于其他现有方法。此外，通过消融实验验证了FTD损失、TV-LPIPS和ADL的有效性，进一步证明了这些方法对提升图像质量的重要性。
在FluxSR的方法中，我们使用了一种叫做FlowTrajectoryDistillation（FTD）的方法来提炼图像超分辨率模型。可以把FTD想象成一种“学习路径”，它帮助我们从一个已经训练好的模型（比如从文本生成图像的模型）中提取出有用的信息，而不需要再使用这个模型进行实时计算。这样，我们就能在生成高分辨率图像时，节省计算资源并提高速度。此外，我们还引入了一些新的损失函数，比如TV-LPIPS和ADL，来确保生成的图像不仅清晰，而且真实感更强。这些技术的结合使得FluxSR能够在一步内生成高质量的图像，同时减少了生成过程中可能出现的伪影和噪声。
