287-B1｜专利图形描述生成，VLM后训练数据策略，视频理解；光学化学结构理解用于分子科学发现

2025-01-25｜IITJodhpur,Microsoft｜AAAI2025｜🟡
http ://arxiv.org/abs/2501.15074v1https ://vl2g.github.io/projects/PatentLMM/
在专利文件中，图形的描述对于知识共享和知识产权的保护至关重要。然而，传统的手动撰写技术描述既费时又繁琐，影响了专利处理和分析的效率。为了解决这一问题，本文提出了一个新的大规模数据集PATENTDESC-355K，包含约355,000个专利图形及其简要和详细的文本描述，数据来源于60,000多份美国专利文件。此外，研究团队还开发了PATENTLMM，一个专门针对专利图形描述生成的多模态大语言模型。该模型的核心在于两个主要组件：PATENTMME，一个专为专利图形设计的多模态视觉编码器，以及PATENTLLAMA，一个经过专利文本微调的LLaMA版本。通过这些创新，研究旨在提高专利图形描述的自动化生成能力，从而促进知识共享和专利文档的快速起草。
本研究的方法分为几个关键步骤，旨在构建一个强大的模型用于专利图形描述生成：
数据集构建：创建PATENTDESC-355K数据集，包含355,000个专利图形及其相应的简要和详细描述，确保数据的多样性和覆盖面。
视觉编码器（PATENTMME）：采用基于LayoutLM的多模态Transformer架构，专门设计来处理专利图形的稀疏布局。该编码器通过结合光学字符识别（OCR）和视觉元素检测，提取图形中的关键元素（如节点、标签和箭头），并通过多种预训练目标（如布局感知掩码建模）来优化模型性能。
语言模型（PATENTLLAMA）：在LLaMA的基础上进行微调，使其适应专利文本的特定语言特征。该模型通过项目层与视觉编码器的输出结合，生成与输入图形相对应的文本描述。
集成模型（PATENTLMM）：将PATENTMME和PATENTLLAMA结合，通过一个投影层实现两者的协同工作，最终生成高质量的描述。
在实验部分，研究团队对PATENTLMM进行了广泛的评估，比较了其在生成简要和详细描述任务中的表现。实验包括对比多个基线模型（如BLIP-2、TextMonkey等），在零-shot和微调设置下进行性能测试。结果显示，PATENTLMM在BLEU、ROUGE和METEOR等指标上均超越了其他模型，尤其在详细描述生成方面表现突出。研究还进行了消融实验，分析了不同组件和预训练目标对模型性能的影响，结果表明，结合多种预训练策略显著提升了模型的描述生成能力。最终，PATENTLMM被证明是专利图形描述生成任务中的一个有效解决方案。
在这项研究中，科学家们开发了一种新的技术，可以帮助自动生成专利图形的描述。他们首先创建了一个包含大量专利图形和相应描述的数据集。接着，他们设计了一个特殊的视觉模型，可以理解这些图形的结构，比如图形中的箭头和标签。同时，他们还使用了一个语言模型，专门训练来生成与专利相关的文本。最后，他们把这两个模型结合在一起，形成一个新的系统，能够根据专利图形自动生成简洁和详细的描述。通过大量实验，他们发现这个新系统的表现比之前的技术要好很多，能够更准确地描述专利图形的内容。这项技术有望提高专利处理的效率，帮助发明者更好地保护他们的创意。
2025-01-20｜NJU,THU,PolyU,JHU,NYU,NVIDIA｜🟡
http ://arxiv.org/abs/2501.14818v1https ://github.com/NVlabs/EAGLE?tab=readme-ov-file
在近年来，开放源代码的视觉语言模型（VLMs）取得了显著进展，逐渐接近商业前沿模型的能力。然而，大多数开放源模型仅发布最终的模型权重，数据策略和实现细节却鲜有披露。本研究从数据中心的视角出发，探讨了VLM后训练的关键作用，强调数据策略在开发前沿VLM中的重要性。通过从零开始构建我们的后训练数据策略，我们分享了开发过程中的详细见解，旨在为开放源社区提供竞争力模型的开发支持。我们引入的数据策略、训练方案和模型设计，成功构建了一系列高效的VLM，命名为Eagle2。其中，Eagle2-9B在多模态基准测试中表现出色，达到了与某些拥有70B参数的竞争模型相媲美的效果。
本研究的方法主要包括以下几个关键部分：
数据策略：
数据收集：采用多样化的数据收集策略，确保数据源的广泛性，涵盖180多个来源。
数据过滤：通过规则基础的过滤机制，去除低质量样本，提升数据集整体质量。
数据选择：构建高质量子集，确保模型训练时使用的数据是最优的。
数据增强：利用多种技术丰富现有数据，提升模型的泛化能力。
模型架构：
采用动态切片和视觉编码器混合（MoVE）设计，确保高分辨率输入的有效处理。
使用SigLIP和ConvNeXt作为视觉编码器，并通过MLP连接器将视觉信息与语言模型有效整合。
训练方案：
三阶段训练：第一阶段对语言和图像模态进行对齐；第二阶段使用多样化的数据进行模型训练；第三阶段则在高质量的视觉指令调优数据集上继续训练。
数据打包：实施平衡的贪婪背包算法，以优化训练过程中的数据利用效率。
在实验部分，我们主要关注Eagle2模型的演变及其性能提升。通过逐步扩大Stage-2数据集的规模，我们发现模型整体表现随着数据量的增加而稳步提升，特别是在包含2M图表、表格和OCR样本时，性能提升最为明显。此外，我们通过引入Stage-1.5预训练阶段，显著提升了模型的基础能力。实验结果显示，经过三阶段训练的Eagle2模型在多个基准测试中超越了初始基线，展现出强大的竞争力。
在我们的研究中，我们采用了一种系统化的方法来构建和训练视觉语言模型。首先，我们收集了大量多样化的数据，确保涵盖各种类型的信息，这样模型就能学习到更丰富的知识。接着，我们对数据进行了严格的筛选，去掉了那些质量不高的样本，以保证训练的数据都是有用的。然后，我们设计了模型的结构，让它能够有效地处理图像和文本信息，确保二者能够顺畅地结合。最后，我们采用了一种三阶段的训练方法，逐步提升模型的性能，确保它在不同类型的任务中都能表现出色。通过这种方式，我们的模型在多个测试中都取得了优异的成绩，展现了其强大的能力。
2025-01-26｜BUAA
http ://arxiv.org/abs/2501.15513v1https ://github.com/ZhangXJ199/TinyLLaVA-Video
TinyLLaVA-Video是一种新型的视频理解模型，其参数不超过40亿，旨在以简单的方式处理视频序列，避免复杂的架构。该模型支持帧率采样和均匀帧采样，具备模块化和可扩展性，能够在有限的计算资源下进行训练和推理。用户可以根据需求替换模型组件。通过实验验证，TinyLLaVA-Video在多个视频理解基准测试中表现出色，其最佳模型的性能与一些现有的70亿参数模型相当。此外，所有代码和训练数据都是公开的，旨在为研究人员提供一个基准，以探索小规模多模态视频理解模型。
TinyLLaVA-Video的设计遵循简单结构和模块化原则，允许用户自定义提取的帧数，支持两种采样方法：帧率采样和均匀帧采样。模型的主要步骤如下：
视觉编码器：使用视觉编码器提取视频帧的特征，生成特征序列。
连接器（重采样器）：在将特征序列输入连接器之前，将其连接为一个长张量，以便进行全局特征提取。
学习查询初始化：通过交叉注意机制，将学习查询初始化为特征序列的整体表示。
训练管道：分为两个阶段：预训练阶段冻结视觉编码器，仅训练连接器以对齐视觉和文本嵌入；微调阶段则微调小规模语言模型和连接器，使用视频-文本对进行多轮对话训练。
在实验部分，TinyLLaVA-Video模型在多个视频理解基准上进行了评估，选择了不同的模型架构和视觉编码器。实验包括：
数据质量影响：通过使用不同质量的数据集进行预训练，观察模型性能的变化。
小规模语言模型效果：比较不同小规模语言模型在相同设置下的表现，发现Qwen2.5-3B在多个基准上表现优异。
视觉编码器比较：评估不同视觉编码器（如SigLIP和CLIP）对模型性能的影响，结果显示SigLIP在大多数基准测试中表现最佳。
采样效果研究：分析不同采样帧数和学习查询数量对模型理解视频信息的影响，发现适当的采样和查询数量能够优化模型表现。
TinyLLaVA-Video的工作原理可以简单理解为一个“视频理解助手”。它首先把视频中的每一帧（图像）送入一个“视觉识别器”，这个识别器会提取出每一帧的特征，就像人眼看到图像时会注意到的颜色、形状等。接着，这些特征会被整合到一起，形成一个完整的特征集合。然后，模型会学习如何根据这些特征理解视频的内容，比如识别动作、场景等。这个过程就像是把视频拆分成许多小的部分，逐一分析，最后再把这些信息汇总起来，从而帮助我们理解视频在讲什么。整个过程非常高效，适合在计算能力有限的情况下使用。
2025-01-26｜THUAIR,PharMolix｜🟡
http ://arxiv.org/abs/2501.15415v1
光学化学结构理解（OCSU）是一项旨在从分子图像中提取化学结构信息并将其转化为可读字符串的任务。这项研究的背景是，在化学及相关领域中，分子结构的可视化信息分散在教科书、期刊和专利中，传统的结构识别方法往往仅关注特定的图像到图表转换，而忽略了更广泛的应用场景。为此，本文提出了OCSU任务，扩展了现有的光学化学结构识别（OCSR）方法，涵盖了从功能组到分子层面再到抽象层面的多级别描述。研究中引入了两个主要的技术路径：基于OCSR的方法和无OCSR的端到端模型，旨在提升分子图像的理解能力，从而促进以分子为中心的科学发现。
在方法部分，研究者提出了两种主要的技术路径以实现OCSU任务：
OCSR基础方法（Double-Check）：
结构图转换：将分子图像转换为SMILES表示，利用OCSR方法进行图形重建。
双重检查机制：通过引入局部模糊原子双重检查，增强原子级别的识别能力，以应对不同绘制风格和图像质量带来的挑战。
特征增强：将局部特征与全局分子特征相结合，通过多层感知机（MLP）进行特征对齐和加权融合，提高识别精度。
无OCSR的端到端模型（Mol-VL）：
视觉编码器：采用视觉变换器（ViT）作为编码器，直接处理分子图像并生成特征嵌入。
大型语言模型（LLM）：将生成的特征输入到LLM中，直接实现从图像到文本的生成，避免了中间步骤的复杂性。
训练策略：通过全参数的监督微调，使用Vis-CheBI20数据集进行训练，优化模型在多任务学习中的表现。
本研究在Vis-CheBI20数据集上进行了广泛的实验，以评估提出方法的有效性。实验内容包括：
性能评估：对比Double-Check与现有OCSR方法的表现，验证其在真实世界专利和期刊文章基准上的优越性。
OCSU任务的执行：评估Mol-VL在功能组描述、分子描述和IUPAC命名等多个子任务上的表现，并与传统OCSR方法进行比较。
鲁棒性测试：在不同的图像扰动条件下，测试方法的稳健性，确保在真实应用场景中的可行性。
在这项研究中，科学家们想要帮助计算机理解化学分子的图像。为了实现这一点，他们提出了两种方法。第一种方法叫做“Double-Check”，它像是给计算机一个双重保险，让它在识别分子时更加准确。具体来说，计算机首先会将分子图像转换成一种特殊的字符串格式，然后再仔细检查每个原子，确保没有遗漏或错误。第二种方法叫“Mol-VL”，它更先进，直接将分子图像输入计算机，计算机会自动生成描述，而不需要中间转换。这种方法使用了一种叫做“视觉语言模型”的技术，就像一个聪明的助手，能够根据图像直接给出文字描述。通过这些方法，研究者希望计算机能更好地帮助科学家们进行药物发现和其他化学研究。
