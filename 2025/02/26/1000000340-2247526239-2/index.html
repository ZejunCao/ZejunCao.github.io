2025-02-18｜KAIST,GoogleDeepMind,GeorgiaTech,LumaAI,ScaledFoundations｜⭐️⭐️
http ://arxiv.org/abs/2502.12632v1
在视频生成领域，MALTDiffusion（Memory-AugmentedLatentTransformers）旨在解决生成任意长度视频的挑战。传统的扩散模型虽然在生成短视频方面表现出色，但通常只能处理2到10秒的片段，无法有效生成长时间的视频内容。MALT通过引入记忆增强机制，能够将长视频分割成多个短片段，并在此基础上进行自回归生成。这种方法使得模型能够在生成过程中保持对长时间上下文的理解，从而生成更连贯和高质量的视频。研究表明，MALT在多个长视频基准测试中超越了现有的最先进技术，展现出强大的生成能力。
MALTDiffusion的核心在于其独特的架构和训练策略，主要包括以下几个方面：
记忆增强机制：MALT通过引入递归注意力层，将多个视频片段编码为一个紧凑的记忆向量。这种机制允许模型在生成新帧时，能够参考之前生成的上下文信息。
自回归生成：模型将长视频分割为多个短片段，逐一进行生成。每次生成新片段时，模型会根据之前生成的片段和记忆向量进行条件生成，从而确保生成内容的连贯性。
训练技术：为了解决长时间生成中的稳定性问题，MALT采用了噪声增强的训练策略，使得生成模型对噪声的鲁棒性增强，降低了错误传播的风险。这种方法在生成过程中能够保持视频质量的一致性。
研究团队通过在多个长视频基准测试上评估MALT的性能，验证了其在长时间上下文理解和稳定性方面的优势。实验结果显示，MALT在UCF-101数据集上生成的128帧视频的Fréchet视频距离（FVD）为220.4，显著优于之前的648.4的最佳结果。此外，在Kinetics-600数据集中，MALT也展现出更好的生成效果。通过对比实验，研究者还探讨了不同训练组件（如递归记忆和长时间稳定性训练策略）对最终生成质量的影响，结果表明这些组件对生成性能的提升至关重要。
MALTDiffusion是一种新的视频生成技术，它的目标是能够生成任意长度的视频。传统的方法往往只能生成短片段，比如几秒钟，而MALT则通过一种叫做“记忆增强”的方法来解决这个问题。简单来说，MALT会把一个长视频分成多个小片段，然后一个接一个地生成这些片段。在生成每个新片段时，MALT会参考之前生成的内容，就像人们在讲故事时，会记得之前发生了什么一样。
为了让MALT更好地工作，它还使用了一些聪明的训练技巧，比如在训练时加入一些噪声，这样可以让模型在生成过程中更稳定，不容易出错。通过这些方法，MALT不仅能生成更长的视频，还能保持视频的质量和连贯性。实验结果显示，MALT在多个测试中表现优异，证明了它的强大能力。
2025-02-18｜UGlasgow,ShandongU,TelefónicaScientificResearch｜⭐️⭐️
http ://arxiv.org/abs/2502.12945v2
本研究探讨了大型语言模型（LLMs）在流行微视频生成中的应用，尤其是如何利用这些模型简化视频创作过程。随着TikTok和YouTube等平台的崛起，微视频已成为一种重要的内容形式，吸引了大量观众并为内容创作者带来了可观的收益。然而，制作受欢迎的微视频通常需要大量的人力和物力资源。为了解决这一问题，本文提出了LLMPopcorn，这是一个利用LLMs协助生成流行微视频的综合性管道。研究的核心问题包括：如何有效利用LLMs来支持微视频生成、如何通过提示增强技术优化生成内容的受欢迎程度，以及不同的LLMs和视频生成器在流行微视频生成任务中的表现。
LLMPopcorn管道的设计包括以下几个关键步骤：
用户提示构建：根据用户输入的主题生成具体和抽象的提示，确保涵盖多样化的内容类型。
视频生成：LLM处理这些提示并生成视频标题和生成提示，将这些提示输入到视频生成模型中以创建相应的视频内容。
受欢迎度评估：使用预训练的离线视频受欢迎度预测模型对生成的视频进行评估，以量化其潜在的受欢迎程度。该模型依据视频的评论数量来判断其受欢迎程度。
提示增强：通过检索相关视频并进行链式思维推理，进一步优化输入提示，以提高生成内容的质量和受欢迎度。
在实验部分，研究者使用Microlens数据集设计了两种用户提示类型（具体和抽象），并生成了200个不同的提示，涵盖五个主要类别：动漫、精致美食、日常分享、影视作品和游戏。通过比较LLMPopcorn生成的视频与人类创作的视频，使用中位数受欢迎度作为评估指标，研究发现LLMPopcorn生成的内容在某些情况下能够达到或超过人类创作的平均水平。此外，实验还评估了不同LLMs（如DeepSeek-V3、Llama-3.3-70B和ChatGPT-4o）和视频生成模型（如LTX-Video和HunyuanVideo）的表现，表明DeepSeek-V3在大多数情况下表现出色。
在我们的研究中，我们提出了一种新方法，叫做LLMPopcorn，旨在帮助人们更轻松地制作受欢迎的短视频。首先，我们会根据用户的需求生成一个视频主题，比如“如何制作美味的印度咖喱”。接下来，利用大型语言模型（LLM），我们会创建一个视频标题和详细的制作提示。然后，这些提示会被送入视频生成程序，自动生成视频内容。为了确保这些视频能够吸引观众，我们还会使用一个预测工具，评估这些视频可能会获得多少评论，帮助我们判断它们的受欢迎程度。通过这种方式，我们希望能大幅简化视频制作过程，让更多人能够轻松创造出受欢迎的内容。
2025-02-18｜StonyBrookU,MetaAI,Meta-CodecAvatarsLab｜⭐️
http ://arxiv.org/abs/2502.13133v1https ://aggelinacha.github.io/AV-Flow/
本研究提出了一种名为AV-Flow的新型生成模型，旨在通过文本输入生成音频-视觉的4D说话头像。当前的人工智能交流多限于文本，缺乏自然的音频和视觉交互，而AV-Flow通过并行的扩散变换器架构，实现了语音与面部动态的同步生成。该模型不仅能合成逼真的语音和唇动，还能生成生动的面部表情和头部动作，增强了人与AI之间的互动体验。此外，AV-Flow还支持在双向对话中使用，能够实时倾听用户的音频-视觉输入并作出反应，促进更具同理心的互动。
AV-Flow的核心方法包括以下几个步骤：
输入表示：模型接收文本输入并通过预训练的语音识别模型提取音频特征，生成相应的音素和面部动态编码。
架构设计：AV-Flow采用两个相互连接的扩散变换器，一个用于音频生成，另一个用于视觉输出。通过中间的高速公路连接，确保音频和视觉之间的有效通信。
音频-视觉融合：音频变换器生成梅尔频谱图，视觉变换器生成头部和面部动态。两者通过上采样和特征融合实现同步。
流匹配训练：模型使用流匹配作为训练目标，优化音频和视觉生成的时间一致性，确保快速推理和高质量输出。
双向对话能力：在双向交互中，模型能够接收用户的音频-视觉信号，指导生成自然的对话头像。
在实验部分，研究团队使用公开数据集进行评估，包括多个参与者的对话数据。通过定量和定性分析，评估AV-Flow在唇同步、真实感、多样性和音频-视觉对齐等指标上的表现。与现有方法进行对比，AV-Flow在多个指标上表现优异，特别是在生成自然的4D说话头像方面。此外，实验还展示了AV-Flow在双向对话中的表现，能够根据用户的输入实时调整头像的表情和动作，增强交互的真实感。
AV-Flow的工作原理可以简单理解为一个“翻译器”，它将输入的文本转换成一个能够说话和表情丰富的3D头像。首先，系统会把文本转化为音频信号和面部动作的指令。然后，这些指令会通过两个相互协作的部分来处理：一个专门生成声音，另一个负责生成头像的面部表情和头部动作。通过这种方式，AV-Flow可以确保声音和面部动作的完美同步，就像真人在说话一样。此外，它还可以在对话中“倾听”用户的声音和表情，并做出相应的反应，使得人与AI的互动更加自然和生动。
2025-02-18｜UAmsterdam｜⭐️
http ://arxiv.org/abs/2502.13034v1
自然语言生成（NLG）在理解和描述视觉内容方面至关重要，尤其是在多图像或视频序列的情况下。本研究探讨了多图像到文本生成的不同任务，例如视频字幕生成、变更字幕生成、视觉故事讲述和多图像/视频问答等。尽管已有研究主要集中在单图像生成上，但随着视频和图像序列数据的增加，研究者们开始关注多图像情境下的文本生成。本文提出了这些任务在处理复杂的视觉和语言关系时所面临的共同挑战，并强调了在这一领域的未来研究方向。通过对不同任务的比较分析，本文旨在为多图像到文本生成的研究提供新的见解，并推动更有效的模型开发。
本研究的核心方法是对多图像到文本生成任务的特点进行定量分析。具体步骤如下：
任务定义与目标：明确每种任务的输入和输出要求，例如视频字幕生成需要将视频帧转化为简洁的文本描述，而视觉故事讲述则需要生成更为复杂的叙述。
挑战识别：识别并讨论在多图像到文本生成中普遍存在的挑战，如实体跟踪、视觉对齐、知识整合和文本连贯性等。这些挑战影响模型的生成质量和准确性。
模型架构评估：回顾不同的模型架构和学习策略，分析它们在处理多模态输入时的有效性，特别是在如何将视觉信息转化为语言描述方面。
未来研究方向：基于分析结果，提出未来研究的建议，如如何改进模型以更好地处理视觉与语言之间的关系，并提高生成文本的质量。
为验证提出的方法，本文对五个相关数据集进行了实验，涉及不同的多图像到文本任务。每个任务从各自的数据集中随机选择100个实例，通过计算视觉相似性和文本一致性分数，对模型在不同任务中的表现进行评估。实验结果显示，任务之间在视觉和文本层面的相似性存在显著差异。例如，变更字幕生成任务的输入图像通常相似，而视觉故事讲述任务中的图像则可能具有较低的相似性。通过对比这些分数，研究者能够更好地理解不同任务的特性和模型生成文本的潜力。
在这项研究中，我们探讨了如何从多个图像或视频生成自然语言文本。首先，我们定义了不同的任务，比如视频字幕生成和视觉故事讲述，明确它们的输入和输出要求。接着，我们识别了在这些任务中可能遇到的一些常见挑战，比如如何跟踪图像中的人物和物体，确保生成的文本与图像内容一致，以及如何将额外的信息整合到生成的文本中。最后，我们回顾了当前的模型架构，提出了一些改进的建议，以帮助未来的研究者更有效地处理这些复杂的视觉和语言关系。通过这些分析，我们希望能够推动多图像到文本生成领域的进一步发展。
