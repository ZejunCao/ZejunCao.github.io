2025-02-20｜THU,SUTD
http ://arxiv.org/abs/2502.14834v1https ://github.com/THU-KEG/LongWriter-V
本研究旨在提升大型视觉语言模型（VLMs）在生成超长文本方面的能力，尤其是在处理基于视觉输入的长文本生成任务时，现有模型的输出长度通常受到限制，难以超过1000个单词。为了解决这一问题，研究者们提出了LongWriter-V-22k数据集，包含22158个样本，涵盖从0到10000字的多种输出需求。此外，研究还引入了迭代直接偏好优化（IterDPO）方法，通过对长输出进行分段处理，利用人类反馈进行迭代修正，以提高生成文本的质量和连贯性。最终，经过训练的模型在新的评估基准MMLongBench-Write上表现出色，超过了其他大型模型如GPT-4o。
研究方法主要包括以下几个步骤：
数据收集：
SFT数据集：使用LongWrite-Agent-V生成长文本，首先生成大纲，然后逐段填充内容，最终合成完整文本。数据来源于MMEvol数据集，经过筛选后得到8115个单图像指令和6313个多图像指令。
DPO数据集：收集VLM模型生成的长文本，并由人类进行修正，形成偏好对，以便用于优化模型输出的质量。
模型训练：
采用两种不同参数的开源VLM作为基础模型，进行监督微调（SFT），以LongWrite-V-22k数据集为训练基础，增强模型对长文本生成的适应性。
优化过程：
通过IterDPO方法，分段处理长输出的修正反馈，提升模型对长文本的生成能力，确保生成内容的连贯性和准确性。
实验部分包括对模型在MMLongBench-Write基准上的评估，主要关注模型的输出长度和质量。通过对比不同模型的表现，发现LongWriter-V模型在生成超过2000字的文本时具有显著优势，且在多个评估指标上均优于其他模型。实验结果显示，LongWriter-V-7B-DPO模型在长度得分和质量得分上均有提升，验证了迭代优化方法的有效性。此外，研究还进行了人类评估，发现LongWriter-V模型在用户偏好上也表现优异。
在这个研究中，科学家们想要让计算机能够更好地生成长篇文章，尤其是当它们需要根据图片来写作时。为了实现这一目标，他们首先收集了大量的例子，包含了从短文到长文的各种写作任务。然后，他们设计了一种新的训练方法，这种方法可以让计算机在写作时，不仅能一次性生成内容，还能在写作过程中逐步改进自己的输出，像是一个老师不断给学生反馈一样。最后，通过对比实验，他们发现这种新方法的效果非常好，计算机能够生成的文本长度和质量都大大提高。
2025-02-20｜CQU,CAS,Kuaishou
http ://arxiv.org/abs/2502.14864v1https ://github.com/Nomothings/CHARGE.git
本研究提出了一种新颖的多模态检索增强生成（MRAG）任务，专注于图表格式的信息处理。当前的MRAG基准主要集中于简单的图像和文本交互，而忽略了复杂的视觉格式如图表，这在现实应用中非常常见。为了解决这一问题，本文引入了图表基础的MRAG（Chart-basedMRAG），并提出了一个名为CHARGE的框架，用于半自动生成高质量的评估样本。该框架通过结构化的关键点提取、跨模态验证和基于关键点的生成，构建了一个全面的图表MRAG基准（Chart-MRAGBench），涵盖了来自八个领域的4738对问答对。这项研究揭示了当前方法在处理图表场景时的局限性，强调了对复杂多模态交互的评估需求。
CHARGE框架的设计旨在自动生成多模态的问答对，采用三阶段的处理流程：
关键点提取：从文本和图表数据中提取自包含的关键点。关键点分为两类：文本关键点（KT）和图表关键点（KC），以捕捉核心信息。
跨模态验证：通过交叉验证机制确保提取的关键点信息的真实性，验证过程由大型语言模型（LLM）执行，以判断信息是否属于其声称的模态。
问答对生成：结合相关的关键点生成多样化的问答对，采用随机选择的关键点作为查询，检索候选关键点，并通过LLM生成最终的问答对。这一过程支持单点问答和多跳问答，要求整合来自不同来源的知识点。
实验部分通过Chart-MRAGBench对多种主流检索方法和大型语言模型（MLLMs）进行了系统评估。我们比较了不同检索策略的性能，包括统一多模态嵌入、组合向量存储和分离向量存储等。结果表明，当前的多模态检索模型在图表场景下表现不佳，尤其是在知识密集型任务中。实验还揭示了现有模型在图表数据检索中的局限性，强调了开发针对图表的专用架构的必要性。最终，我们通过引入准确性和覆盖率这两个新指标，全面评估了模型在图表基础MRAG任务中的表现。
CHARGE框架的工作原理可以简单理解为一个三步走的过程。首先，我们从文本和图表中提取出重要的信息点，这些信息点就像是每段文字或每个图表中的关键数据。接下来，我们需要确认这些信息点是否准确地对应于它们的来源，比如确保一个图表中的数据确实反映了文本中的描述。最后，我们将这些信息点组合在一起，生成相关的问题和答案。这就像是把拼图的各个部分组合成一幅完整的画面。通过这种方式，CHARGE能够帮助我们更好地理解和回答复杂的多模态问题，尤其是当这些问题涉及到图表时。
2025-02-20｜CASIA,UCAS,AlibabaGroup,BJTU,ShanghaiTech
http ://arxiv.org/abs/2502.14282v2https ://github.com/X-PLUG/MobileAgent/tree/main/PC-Agent
PC-Agent是一个针对复杂任务自动化的分层多代理协作框架，专为个人计算机（PC）场景设计。与智能手机相比，PC环境具有更复杂的交互特性和更繁琐的应用间工作流，因此需要更强大的自动化能力。PC-Agent通过引入主动感知模块（APM）来增强对屏幕内容的理解，并采用分层的多代理协作结构来有效处理复杂用户指令和相互依赖的子任务。该框架的核心设计包括指令分解、进度跟踪、动态决策反馈等，旨在提升任务成功率。实验证明，PC-Agent在处理复杂任务时的成功率显著高于现有方法，展示了其在PC自动化领域的广泛应用潜力。
PC-Agent的设计包含以下几个关键组成部分：
主动感知模块（APM）：
利用无障碍树提取屏幕上交互元素的位置和意义。
结合意图理解代理和OCR工具，精确提取文本内容及其位置。
分层多代理协作架构：
管理代理（MA）：负责将用户指令分解为参数化的子任务，并管理子任务间的通信。
进度代理（PA）：跟踪每个子任务的执行进度，确保信息的准确传递。
决策代理（DA）：根据感知信息和进度信息，逐步做出执行决策。
反思基础的动态决策机制：
设置反思代理（RA）来监控执行结果，及时反馈并调整决策，确保任务执行的准确性。
这种分层结构和动态反馈机制使得PC-Agent能够有效应对复杂的用户指令和交互环境。
为验证PC-Agent的有效性，研究团队创建了一个名为PC-Eval的新基准测试，包含25个复杂的用户指令，涉及8个常用的PC应用程序。实验结果表明，PC-Agent在完成复杂任务时的成功率达到56.0%，而子任务成功率则高达76.0%。与现有的单一代理方法（如UFO和Agent-S）相比，PC-Agent在任务成功率上有显著提升，展示了其在处理复杂指令方面的优势。此外，通过消融实验，研究人员进一步证实了主动感知模块和管理代理在提升任务执行效率和准确性方面的重要性。
PC-Agent的工作原理可以想象成一个团队协作的过程。首先，主动感知模块就像一个侦察员，它负责观察电脑屏幕，识别出各种按钮和文本的位置和意义。接下来，管理代理就像一个项目经理，它会把复杂的任务分解成更小的部分，确保每个部分都能顺利进行。然后，进度代理会跟踪每个子任务的进展，确保每个步骤都在按计划进行。最后，决策代理就像一个执行者，它根据前面的信息逐步做出决策，确保任务能够顺利完成。如果在执行过程中出现错误，反思代理会及时发现并反馈给决策代理，帮助它调整策略。这种团队合作的方式，使得PC-Agent能够高效地处理复杂的指令和操作。
2025-02-20｜UPenn,AllenAI
http ://arxiv.org/abs/2502.14846v1https ://yueyang1996.github.io/cosyn/
在文本丰富的图像理解领域，传统的视觉语言模型（VLMs）面临数据稀缺的挑战。为了解决这一问题，研究者们提出了CoSyn框架，它利用文本生成能力强大的大型语言模型（LLMs）来自动生成合成的多模态数据。通过输入描述特定领域的文本（如“营养成分标签”），CoSyn能够生成相应的代码（Python、HTML、LaTeX等）来渲染合成图像，并将这些图像与文本指令结合，创建出高质量的训练数据。使用CoSyn生成的数据集包含40万张图像和270万行视觉语言指令，经过全面的实验验证，该模型在多个基准测试中表现优异，超越了许多竞争对手，包括开源模型和一些私有模型。
CoSyn框架的核心在于其灵活的合成数据生成流程，具体步骤如下：
输入查询：用户提供关于所需图像类型的文本输入，例如“生成一本书的封面”。
主题生成：根据用户输入，CoSyn选择合适的生成管道，并生成与特定主题相关的多样化内容。
数据内容生成：系统生成详细的数据内容，准备转换为代码。
代码生成：将生成的数据内容转换为可执行代码，以渲染合成图像。
指令生成：使用生成的代码作为上下文，提示LLM生成相应的文本指令，包括问题、答案和推理解释。
CoSyn支持多种渲染工具（如Matplotlib、HTML、LaTeX等），并通过集成20个生成管道来确保生成的图像多样性和质量。该系统还利用“角色”概念来增强生成内容的多样性，使得合成数据更具代表性。
在实验部分，研究者们验证了CoSyn生成的合成数据在视觉语言模型训练中的有效性。实验设置包括：
模型架构：采用与现有模型相似的图像预处理和架构，使用MLP层连接视觉编码器和预训练的LLM。
训练过程：模型通过在合成数据上进行监督微调，评估其在七个文本丰富基准上的表现。
基线比较：与多种开源和私有模型进行比较，分析合成数据对模型性能的提升。
结果显示，使用CoSyn生成的合成数据显著提高了模型在新任务上的泛化能力，尤其是在面对未见过的任务时，其表现优于传统模型。
CoSyn的工作原理可以简单理解为一个“智能助手”，它根据用户的需求生成图像和相关文本。首先，用户告诉系统他们想要什么样的图像，比如一本书的封面。接着，系统会想出一个主题并生成一些内容，比如书名和作者。然后，它会把这些内容转化为代码，像是给计算机下达指令去画图。最后，系统会根据这些图像生成一些问题和答案，帮助模型学习如何理解这些图像。通过这种方式，CoSyn不仅能生成各种各样的图像，还能提供丰富的文本信息，使得模型在理解图像时更加准确和高效。
2025-02-20｜HUJI,UChicago
http ://arxiv.org/abs/2502.14789v1
在计算机视觉和图形学领域，随着对三维场景理解和编辑能力的需求不断增加，研究者们提出了一种新方法，该方法通过结构性解耦特征场的蒸馏，来提升三维场景的理解和编辑能力。该方法利用预训练的2D特征，通过将其转化为三维特征，从而实现对三维场景的语义分割、反射处理、颜色编辑以及粗糙度调整等多种操作。研究表明，传统的2D特征在转化为3D特征时存在物理不一致性，而本研究通过引入多种解耦特征场，能够有效捕捉视图依赖和视图独立的特征，显著提升了三维理解和编辑的性能。
本研究的方法分为几个主要步骤：
特征场的解耦：通过两个多层感知机（MLP）模型，分别提取视图独立特征（f_indep）和视图依赖特征（f_ref）。视图独立特征仅依赖于3D点的位置，而视图依赖特征则考虑观察角度。
物理属性建模：将场景的颜色分解为视图独立颜色和视图依赖反射颜色。通过对颜色和密度的建模，实现高质量的体积渲染。
训练目标：使用重建损失和正则化损失来训练模型，确保生成的特征与真实图像之间的匹配。
分割与编辑：通过选择特定的特征组件，进行3D分割和编辑。用户可以通过交互选择区域，以控制物理属性，如颜色、反射和粗糙度。
在实验部分，研究者使用了多个数据集，包括合成场景和真实场景，评估方法在3D语义分割和编辑任务上的表现。实验结果显示，与现有方法相比，所提方法在物体分割和反射处理方面具有显著优势。特别是在处理反射区域时，能够有效分离和编辑反射特征，提升了操作的灵活性和准确性。通过与其他基线模型的比较，结果表明该方法在各项任务中均表现出色，展示了其在实际应用中的潜力。
在这项研究中，科学家们开发了一种新技术，帮助计算机更好地理解和编辑三维场景。想象一下，你在玩一个3D游戏，想要改变某个物体的颜色或表面光滑度。这项技术的核心是将图片中的信息转化为三维模型，让计算机可以单独处理每个特征，比如物体的颜色和反射光。研究者使用了一种叫做“解耦”的方法，将这些特征分开处理，就像把一个复杂的拼图拆成几个简单的部分。这样，用户就可以更灵活地调整场景中的每个元素，比如改变一个球的颜色而不影响它的反射效果。这种方法在多个实验中表现优异，显示了它在游戏、动画和虚拟现实等领域的广泛应用前景。
2025-02-20｜TUMunich
http ://arxiv.org/abs/2502.14684v1https ://github.com/zqlin0521/cdgs-release
本文提出了一种新的方法，称为CDGS（Confidence-AwareDepthRegularizationfor3DGaussianSplatting），旨在提升3DGaussianSplatting（3DGS）在三维重建中的几何准确性。尽管3DGS在新视角合成（NVS）方面表现出色，具有快速渲染和高质量结果的优势，但其在几何重建中的准确性仍然受到限制，主要是由于缺乏明确的几何约束。CDGS通过结合单目深度估计的多线索置信度图和稀疏的结构光束法（SfM）深度信息，动态调整优化过程中的深度监督，从而提高了几何细节的保留能力，并在NVS质量和几何准确性上达到了竞争水平。实验结果表明，CDGS在公共基准数据集TanksandTemples上显示出更稳定的收敛行为和更准确的几何重建结果，PSNR提高了2.31dB，并在M3C2距离度量中实现了一致的较低几何误差。
CDGS的核心方法包括两个主要组成部分：深度精炼与对齐，以及置信度感知的深度正则化。具体步骤如下：
深度精炼与对齐：
首先，利用预训练的单目深度估计模型，从输入图像中获得初步深度图。
接着，通过将初步深度图与来自SfM的稀疏深度数据进行对齐，使用梯度下降方法优化深度值，从而提供几何约束。
置信度感知深度正则化：
生成置信度图，通过分析深度图及其对应RGB图像的特征，评估每个深度值的可靠性。
在优化过程中，结合局部深度可靠性评估与全局对齐质量，动态调整深度损失权重，以确保优化过程的稳定性和准确性。
通过以上步骤，CDGS有效整合了深度信息，增强了3DGS的优化过程，确保了几何结构的稳定性。
实验部分使用TanksandTemples基准数据集进行评估，涵盖了新视角合成质量和几何重建准确性。实验设置包括：
数据集：选择包含多视角图像和高精度激光扫描地面真值几何的复杂场景。
评估指标：使用PSNR、SSIM和LPIPS等2D指标评估图像合成质量，同时采用F-score、几何完整性和M3C2距离度量等3D指标评估几何重建准确性。
实现细节：在NVIDIATeslaV100上实现，使用PyTorch框架进行训练，设置了适当的学习率和收敛阈值。
结果表明，CDGS在多个场景中表现出色，尤其是在几何细节保留和重建准确性方面，与原始3DGS和其他先进方法相比，显示出明显的优势。
在CDGS方法中，我们首先从一张图片中获取初步的深度信息，就像给每个像素加上一个“深度标签”。然后，我们会把这些深度标签和从其他视角获得的深度信息进行比对和调整，确保它们尽可能准确。这一过程就像是在拼图时，把每块拼图放到合适的位置。接下来，我们使用一个“信心评分”系统，来判断哪些深度标签更可靠。这个评分会影响我们在优化时的关注点，确保我们在训练过程中更加注重那些更靠谱的深度数据。通过这种方式，我们不仅可以更好地理解三维空间的结构，还能提高我们生成的三维图像的质量和准确性。
