2025-02-21｜WHU,PolyU
http ://arxiv.org/abs/2502.15393v1
本文探讨了大型多模态模型（LMMs）在生成长文本描述方面的能力，特别是针对视频的长字幕生成任务。尽管LMMs在处理长视频输入方面表现出色，但在生成相应长度的丰富输出时仍面临挑战。通过一系列控制实验，研究发现训练数据中长字幕样本的稀缺是限制模型输出长度的主要因素。为了解决这一问题，本文提出了LongCaption-Agent框架，通过聚合多层次描述来合成长字幕数据，并构建了新的长字幕数据集LongCaption-10K。此外，还开发了LongCaption-Bench基准，用于全面评估LMMs生成长字幕的质量。实验表明，通过将LongCaption-10K纳入训练，LMMs能够生成超过1000字的长字幕，同时保持高质量的输出。
本文提出的方法主要包括以下几个步骤：
多层次描述合成：LongCaption-Agent框架通过三个层次生成长字幕：帧级、片段级和视频级。首先，使用LMM从每个采样帧中提取静态细节信息；其次，将视频分割为多个片段，利用LMM提取每个片段的时序信息；最后，利用LLM将帧级和片段级描述整合为完整的长字幕。
数据集构建：基于LongCaption-Agent框架，构建了包含10,000个长字幕样本的LongCaption-10K数据集。该数据集涵盖了不同时长的视频，并通过多样化的提示词确保模型的泛化能力。
基准测试：开发了LongCaption-Bench基准，包含281个测试视频，每个视频的平均时长为1060.4秒，平均字幕长度为1161.3字。该基准用于评估LMMs在生成长字幕时的长度、质量和相关性。
实验部分通过对比开源模型和专有模型在LongCaption-Bench上的表现，验证了所提方法的有效性。实验结果表明，经过LongCaption-10K训练的模型在输出长度、质量和相关性方面均优于其他模型，甚至超过了某些专有模型。此外，通过消融实验，进一步验证了视觉上下文窗口扩展和长字幕数据集对模型性能的提升作用。定性分析也展示了所提模型在生成详细且连贯的长字幕方面的优势。
这篇文章主要研究如何让AI模型生成更长的视频描述。虽然现在的AI模型可以处理很长的视频，但它们生成的描述通常比较短，无法涵盖视频中的所有细节。为了解决这个问题，作者提出了一个叫LongCaption-Agent的框架。这个框架通过三个步骤来生成长描述：首先，从视频的每一帧中提取细节信息；然后，将视频分成多个片段，提取每个片段的时间信息；最后，将这些信息整合成一个完整的长描述。为了训练模型，作者还创建了一个包含10,000个长描述的数据集，并开发了一个测试基准来评估模型的表现。实验结果显示，经过训练的模型能够生成超过1000字的长描述，而且质量也很高。
2025-02-17｜YonseiU,NCSOFTCorporation｜NAACL2025
http ://arxiv.org/abs/2502.14892v1https ://jun297.github.io/EgoSpeak/
本文介绍了一种名为EgoSpeak的创新框架，旨在解决在真实环境中预测何时开始对话的基本挑战。EgoSpeak通过从说话者的第一人称视角建模对话，专门为需要持续观察环境并动态决定何时说话的对话代理设计。该框架整合了四个关键能力：第一人称视角、RGB处理、在线处理以及未剪辑视频处理。此外，本文还提出了YT-Conversation，一个从YouTube收集的多样化对话视频集合，用于大规模预训练。实验结果表明，EgoSpeak在EasyCom和Ego4D数据集上优于随机和基于沉默的基线方法，并强调了多模态输入和上下文长度在有效决定何时说话中的重要性。
EgoSpeak框架的核心在于其四个关键能力：
第一人称视角：通过从佩戴者的视角捕捉视频，模拟真实世界中的对话代理如何感知和参与动态的多说话者环境。
RGB处理：处理RGB视频帧，以捕捉视觉信息，特别是在音频或非语言线索不可靠的情况下。
在线处理：实时处理视频流，使对话代理能够动态决定何时说话，从而实现更自然和流畅的对话。
未剪辑视频处理：处理包含沉默和零星互动的连续视频流，以捕捉对话中的完整上下文。
此外，EgoSpeak还引入了YT-Conversation数据集，该数据集包含从YouTube收集的多样化对话视频，用于大规模预训练。通过整合这些能力，EgoSpeak能够更有效地处理真实世界对话的复杂性。
实验在EasyCom和Ego4D两个数据集上进行，验证了EgoSpeak在各种对话上下文中的有效性。实验结果表明，EgoSpeak在实时预测说话时机方面优于随机和基于沉默的基线方法。此外，实验还分析了多模态输入和上下文长度对预测效果的影响，发现结合音频和视觉输入能够显著提高预测准确性。实验还引入了YT-Conversation数据集，用于大规模预训练，进一步提升了模型的性能。
EgoSpeak的核心思想是让对话代理像人类一样，通过观察周围环境来决定何时说话。它通过佩戴者的视角捕捉视频，模拟真实世界中的对话场景。EgoSpeak不仅处理视频中的视觉信息，还结合音频信息，以便在对话中更准确地判断何时该说话。此外，它还能实时处理视频流，确保对话代理能够快速响应环境变化。为了训练这个系统，研究人员还从YouTube上收集了大量的对话视频，帮助模型学习如何在各种复杂情况下做出最佳决策。实验证明，EgoSpeak在预测说话时机方面比随机猜测或简单的沉默检测方法更有效，尤其是在结合了视觉和音频信息的情况下。
2025-02-21｜TUMunich,MCML,NVIDIA
http ://arxiv.org/abs/2502.15480v1https ://florianhofherr.github.io/neural-brdfs
本文对当前最先进的神经双向反射分布函数（BRDF）建模方法进行了全面比较。BRDF是捕捉光与物质复杂交互的重要工具，近年来，多种神经方法被用于BRDF建模，包括基于现有参数模型的方法和纯神经参数化方法。尽管这些方法都取得了令人印象深刻的结果，但文献中缺乏对这些方法的全面比较。本文通过定性和定量重建质量的分析，以及对互易性和能量守恒的评估，提出了两种扩展方法：一种用于将反射分为漫反射和镜面反射的神经BRDF的加法组合策略，另一种通过构造确保互易性的输入映射。
基于参数模型的神经BRDF：这类方法依赖于物理基础的BRDF模型，神经网络预测反射模型的参数，然后使用这些参数和视角、光线方向计算BRDF值。本文评估了基于Torrance-Sparrow模型和DisneyBRDF的神经BRDF。
纯神经BRDF：这类方法不依赖于参数模型，而是直接使用神经网络预测BRDF值。本文评估了使用单个MLP预测BRDF值的方法，以及将反射分为漫反射和镜面反射的加法组合方法。
角度参数化和输入编码：为了处理高频率数据，本文使用了Rusinkiewicz角度和位置编码来增强输入数据的表示。
确保互易性的新映射：本文提出了一种新的角度映射方法，通过构造确保BRDF的互易性，而之前的方法仅通过软约束来确保。
增强加法分割：本文提出了一种新的扩展方法，通过预测一个权重来减少漫反射部分，从而更好地模拟物理行为。
本文在基于MERL的半合成数据集和DiLiGenT-MV真实世界数据集上进行了实验。实验结果表明，纯神经方法在高度镜面材料上具有显著优势，而在真实世界数据集上，各方法之间的差异较小。本文还分析了各方法在能量守恒和互易性方面的表现，发现纯神经方法在真实世界数据上难以从数据中学习互易性。本文提出的新映射方法通过构造确保了互易性，而增强加法分割方法则提高了现有方法的性能。
通俗易懂：这篇文章主要研究了如何用神经网络来模拟光线与物体表面的交互，也就是BRDF。BRDF描述了光线从不同角度照射到物体表面后，反射到不同方向的光的强度。文章比较了几种不同的方法，包括基于物理模型的方法和纯神经网络的方法。基于物理模型的方法使用神经网络来预测一些已知物理模型的参数，然后通过这些参数计算BRDF值。而纯神经网络的方法则直接使用神经网络来预测BRDF值，不需要依赖物理模型。文章还提出了一些改进方法，比如通过一种新的角度映射来确保BRDF的对称性，以及通过预测一个权重来更好地模拟漫反射和镜面反射的物理行为。实验结果表明，纯神经网络的方法在处理复杂反射材料时表现更好，但在真实世界的数据上，各方法之间的差异不大。文章还发现，纯神经网络的方法在真实世界数据上难以学习到BRDF的对称性，而提出的新映射方法则通过构造确保了对称性。
2025-02-20｜Stanford,ETHZurich,MicrosoftSpatialAILab
http ://arxiv.org/abs/2502.15011v1https ://sayands.github.io/crossover
CrossOver是一种创新的跨模态3D场景理解框架，旨在通过灵活的、场景级别的模态对齐来解决现有方法在处理多模态3D数据时的局限性。传统方法通常假设所有模态数据都是完全对齐的，而CrossOver则通过构建一个统一的、模态无关的嵌入空间，能够在模态数据不完整或未对齐的情况下实现场景理解。该框架支持多种模态（如RGB图像、点云、CAD模型、平面图和文本描述）的对齐，并通过多阶段训练流程和维度特定的编码器，实现了强大的场景检索和对象定位能力。实验结果表明，CrossOver在ScanNet和3RScan数据集上表现优异，展示了其在现实世界应用中的适应性和灵活性。
CrossOver的核心方法包括以下几个关键部分：
维度特定编码器：针对不同模态的维度（1D、2D、3D），CrossOver设计了专门的编码器。这些编码器能够高效地提取每种模态的特征，而无需依赖显式的3D场景图或语义标签，从而避免了传统方法中对语义一致性的依赖。
多阶段训练流程：CrossOver的训练流程分为三个阶段。首先，对象级别的嵌入捕捉细粒度的模态关系；其次，场景级别的训练构建统一的场景表示，而不需要所有对象对都对齐；最后，维度特定的编码器生成语义无关的跨模态嵌入，使得模型能够在没有语义信息的情况下进行跨模态对齐。
跨模态行为的涌现：尽管CrossOver并未在所有模态对上进行显式训练，但它能够学习到跨模态的关联行为。例如，模型能够识别图像模态中的场景与平面图模态中的场景之间的对应关系，即使这些模态对在训练中并未同时出现。
通过这些方法，CrossOver能够在模态数据不完整或未对齐的情况下，实现跨模态的场景检索和对象定位，为现实世界中的3D场景理解提供了灵活的解决方案。
CrossOver在ScanNet和3RScan数据集上进行了广泛的实验评估，涵盖了跨模态实例匹配、场景检索以及时间序列场景匹配等多个任务。实验结果表明，CrossOver在跨模态实例匹配和场景检索任务中显著优于现有的多模态方法，如ULIP-2和PointBind。特别是在场景检索任务中，CrossOver能够在不同模态之间准确匹配场景，展示了其强大的跨模态对齐能力。此外，CrossOver在处理时间序列场景时也表现出色，能够有效应对场景中对象的移动和变化。实验还验证了CrossOver在模态数据不完整情况下的鲁棒性，进一步证明了其在现实世界应用中的潜力。
CrossOver的核心思想是通过一种灵活的方式，将不同形式的数据（如图片、点云、CAD模型等）对齐到一个统一的空间中，从而让计算机能够更好地理解3D场景。传统的方法通常要求这些数据必须完全对齐，而CrossOver则不需要，它能够在数据不完整或未对齐的情况下，依然找到不同数据之间的关联。
具体来说，CrossOver使用了三种不同的编码器，分别处理1D（如文本）、2D（如图片）和3D（如点云）数据。这些编码器能够高效地提取每种数据的特征，而不需要依赖复杂的语义信息。然后，通过多阶段的训练，CrossOver逐步构建出一个统一的“嵌入空间”，使得不同形式的数据能够在这个空间中找到对应的关系。
最有趣的是，CrossOver能够在没有显式训练的情况下，自动学习到不同数据之间的关联。比如，它能够识别出一张图片中的场景与一个平面图中的场景是同一个地方，即使这两种数据在训练中从未同时出现过。
通过这些技术，CrossOver能够在现实世界中处理复杂的3D场景，比如在虚拟现实、增强现实和机器人导航等领域，提供更加灵活和强大的解决方案。
2025-02-20｜ShanghaiAILab,THU
http ://arxiv.org/abs/2502.14938v1
本文提出了GS-Cache框架，旨在解决大规模3D高斯溅射（3DGS）模型在消费级设备上实现实时高保真渲染的挑战。3DGS模型在虚拟现实（VR）等应用中具有巨大潜力，但其计算和内存需求随着场景规模的扩大而显著增加。GS-Cache通过引入一个以缓存为中心的渲染管道，消除了冗余计算，并结合了效率感知的多GPU调度器和优化的CUDA内核，显著提升了渲染性能。实验表明，GS-Cache在2K双目渲染中实现了高达5.35倍的性能提升，35%的延迟降低，以及42%的GPU内存使用减少，支持超过120FPS的高质量渲染。
GS-Cache框架的核心方法包括以下几个方面：
缓存为中心的渲染管道：通过动态缓存策略，减少连续帧之间的冗余计算。缓存存储前一帧的数据，加速当前帧的渲染。
多GPU弹性调度器：根据渲染帧率的变化，动态分配GPU资源，确保稳定的高帧率渲染。
优化的CUDA内核：针对渲染管道中的计算瓶颈，设计了专用的CUDA内核，进一步提升了渲染性能。
双目立体去冗余：通过合并双目相机的视场，减少立体渲染中的计算冗余，提升渲染效率。
实验在MetaQuest3头戴式显示器上进行，使用MatrixCity数据集构建了城市和街道场景。实验结果表明，GS-Cache在单GPU和多GPU环境下均显著提升了渲染性能。在城市场景中，优化后的渲染管道平均帧率提升了2倍，而在街道场景中，帧率提升更为显著，达到了3.74倍。此外，GS-Cache在保持高质量渲染的同时，显著降低了GPU内存使用和渲染延迟，满足了VR应用的高帧率需求。
GS-Cache框架的核心思想是通过缓存和优化计算来加速3D场景的渲染。想象一下，你在玩一个VR游戏，游戏中的场景非常复杂，渲染每一帧都需要大量的计算。GS-Cache通过记住前一帧的计算结果，避免重复计算，从而加快了渲染速度。此外，它还智能地分配多个GPU的计算任务，确保游戏画面流畅不卡顿。最后，GS-Cache还针对渲染过程中的瓶颈进行了专门的优化，进一步提升了性能。通过这些方法，GS-Cache能够在复杂的3D场景中实现高质量的实时渲染，让你在VR世界中获得更流畅的体验。
