286-A3｜艺术网格生成，室内场景纹理生成，近二阶收敛3DGS，自动驾驶场景理解与生成

主编｜庄才林（CailinZhuang）技术支持｜胡耀淇（YaoqiHu）发布日期｜2025年1月27日｜周一
2025-01-24｜NTU,TencentHunyuan,PolyU,HKUST｜⭐️⭐️
http ://arxiv.org/abs/2501.14317v2https ://nautilusmeshgen.github.io
Nautilus是一种新型的局部性感知自编码器，旨在实现可扩展的艺术风格网格生成。该方法通过建模和利用网格的局部特性，显著提升了生成质量和细节复杂度。传统的自动网格生成方法往往依赖中间表示，缺乏连续表面质量，导致生成的网格在结构上不够精细。Nautilus通过引入创新的标记算法，保持面之间的邻近关系，极大地压缩了序列长度，并实现了高达5000个面的网格生成。此外，Nautilus还采用了双流点条件器，提供多尺度几何指导，确保全局一致性和局部结构的完整性。实验结果表明，Nautilus在保真度和可扩展性方面显著优于现有的最先进方法。
Nautilus的核心方法包括以下几个方面：
Nautilus风格网格标记算法：该算法将艺术创作的网格转化为序列，确保邻近面的关系得以保留，从而提高局部依赖性建模的有效性。
双流点条件器：此机制提供全局和局部几何指导，利用输入点云的信息来生成结构一致的网格。全局编码器提取整体几何特征，而局部编码器则捕捉细粒度的几何信息，确保生成的网格在细节上保持高保真度。
自回归序列解码器：通过自回归方式生成网格，模型预测下一个网格标记的条件概率，确保生成过程的连贯性和一致性。
在实验部分，Nautilus被训练于一个包含311K高质量艺术网格资产的数据集。实验结果展示了Nautilus在多个指标上超越了现有的生成方法，包括生成的网格的结构完整性和细节保真度。在点云条件下，Nautilus生成的网格不仅在视觉上优于其他方法，还在复杂拓扑结构的生成上表现出色。此外，针对单视图图像的条件生成，Nautilus也展现了强大的能力，能够生成与输入图像高度一致的细致网格。用户研究表明，Nautilus的生成结果在用户满意度上也远超其他方法。
在Nautilus的工作流程中，首先我们需要将艺术家创作的网格转化为一种可以计算机理解的序列。这就像把一个复杂的图形拆解成简单的步骤，方便后续的处理。接下来，Nautilus会使用一个双流的条件器来分析输入的点云或图像，从中提取出整体和局部的几何特征。想象一下，整个过程就像是一个建筑师在设计建筑时，既要考虑整体的外观，又要关注每个细节的搭配。最后，Nautilus会通过一种叫做自回归的方式逐步生成网格，确保每一步都是在前一步的基础上进行的，从而最终形成一个完整、复杂且美观的3D网格。这样的方法不仅提高了生成的质量，还能处理更多面数的网格，展现出更丰富的细节。
2025-01-22｜NTU,Alibaba-NTU,HKUST(GZ)｜⭐️🟡
http ://arxiv.org/abs/2501.13969v1
在室内场景的3D纹理生成中，生成高质量的纹理对于室内设计、游戏及增强/虚拟现实（AR/VR）等应用至关重要。尽管近年来3D生成模型的进步促进了内容创作，但在广泛的通用性和多视角风格一致性方面仍然面临挑战。现有方法如2D扩散模型在适应3D纹理生成时，通常存在处理时间长和视觉伪影的问题，而基于3D数据的方法则常常缺乏有效的泛化能力。为了解决这些问题，本文提出了一种名为InsTex的两阶段架构，旨在使用多样化的提示或图像指导生成高质量且风格一致的室内场景纹理。该方法通过深度到图像的扩散先验，采用粗到精的管道，首先生成多视角图像，然后对纹理进行细化，以确保一致性。
InsTex的核心是一个两阶段的纹理生成流程，主要包括以下几个步骤：
场景对象分解：将室内场景网格分解为单独的对象，以便进行对象级的纹理处理。每个对象被转换到规范空间，以便进行后续处理。
粗纹理生成：利用预训练的深度到图像扩散模型，从多个预定义视角生成高质量的2D图像。这些图像被反投影到3D对象的纹理空间，形成初始纹理图。
纹理细化：在细化阶段，使用动态视图分区策略对生成的纹理进行修复与增强，填补缺失区域并提高整体纹理质量。通过引入全局样式图像，确保在不同对象之间保持风格一致性。
场景重组与后处理：将所有纹理化的对象重新组合成完整的室内场景，并进行后处理，以进一步消除伪影并提高整体一致性。
在实验中，InsTex的性能通过与当前最先进的方法进行比较来评估。我们使用3D-FRONT数据集的纹理化网格进行测试，针对每个场景使用两个不同的文本提示。评估指标包括CLIP评分和Inception评分，以测量生成图像的质量和多样性。用户研究也被引入，以评估生成纹理的视觉质量和与文本提示的符合度。结果显示，InsTex在纹理生成的质量和与提示的符合度方面均优于对比方法，且处理时间显著减少，展示了其在室内场景纹理生成中的有效性。
InsTex的工作流程可以简单理解为一个两步走的过程。首先，我们把一个房间的3D模型拆分成不同的家具和装饰物，然后对每个物件进行纹理处理。我们使用一种叫做“扩散模型”的技术，从多个角度生成物件的图片，这些图片就像是我们从不同位置看到的样子。接着，我们把这些图片贴到物件上，确保每个物件的样子都很漂亮且风格一致。最后，我们把所有的物件放回到房间里，并进行最后的调整，确保整个房间看起来和谐且没有任何瑕疵。这样，我们就能够快速生成高质量、风格一致的室内场景纹理，适用于各种设计和游戏应用。
2025-01-22｜UUtah,ZJU,UCLA｜⭐️🟡
http ://arxiv.org/abs/2501.13975v2
本研究提出了一种名为3DGS2的近二阶收敛训练算法，用于提升三维高斯渲染（3DGaussianSplatting,3DGS）的效率和效果。3DGS通过一系列高斯核来表示三维场景，能够实现高质量的图像渲染。然而，传统的基于随机梯度下降（SGD）的方法在训练效率上存在瓶颈，通常需要数分钟甚至更长时间。相较之下，3DGS2通过引入局部牛顿优化策略，显著加快了训练速度，达到10倍以上的提升，同时保持或超越了现有方法在图像质量方面的表现。研究结果表明，3DGS2可以在几秒内完成复杂场景的重建，极大地推动了三维场景合成和重建技术的发展。
3DGS2的核心在于局部牛顿优化的设计，主要包括以下几个步骤：
参数分解：将高斯核的多个属性（如位置、透明度、方向和缩放）分解为独立的子问题，允许每个参数独立优化。
小规模牛顿系统构建：为每个高斯核的参数组建立小规模的牛顿系统，通过并行计算在GPU上高效求解。
局部优化：每个参数的更新通过局部牛顿法进行，避免了全局Hessian矩阵的计算，降低了计算复杂度。
空间关系利用：在多个输入图像之间利用空间关系，减少优化过程中的振荡现象，提高收敛速度。
迭代更新：通过每次迭代更新参数，逐步减小损失函数，最终达到图像重建的最佳效果。
在实验中，研究团队使用多种3DGS数据集对3DGS2算法进行了评估。与传统的SGD方法相比，3DGS2在训练时间上实现了显著的缩短，训练速度提升了10倍以上。同时，在图像质量指标（如结构相似性指数SSIM、峰值信噪比PSNR和感知相似性LPIPS）上，3DGS2的表现同样优于现有的主流算法。实验结果表明，3DGS2不仅能够快速完成复杂三维场景的重建，而且在图像质量上也达到了领先水平，展示了其在实际应用中的潜力和优势。
在这项研究中，科学家们提出了一种新方法，叫做3DGS2，目的是让计算机更快地生成三维图像。传统的方法像是在爬山，慢慢向上走，花费大量时间，而3DGS2则像是找到了一个更陡的坡道，可以更快地到达山顶。这个新方法通过将复杂的问题拆分成小块，分别解决每一块，避免了大规模计算带来的麻烦。它还利用了不同图像之间的关系，这样在优化时就能更有效率，减少了错误。总之，3DGS2让计算机在几秒钟内就能生成高质量的三维图像，比以前的方法快得多，效果也更好。
2025-01-24｜HUST,MEGVI,MachDrive,HKU｜⭐️🟡
http ://arxiv.org/abs/2501.14729v1https ://github.com/LMD0311/HERMES
本文提出了一种名为HERMES的统一自驾世界模型，旨在同时实现3D场景理解和未来场景生成。当前的驾驶世界模型（DWM）通常专注于场景生成，缺乏对环境的理解和推理能力，这限制了其在自主驾驶中的应用。HERMES通过整合鸟瞰视图（BEV）表示，能够有效地捕捉多视角空间信息，保持几何关系和交互作用。该模型引入了世界查询的概念，通过将世界知识融入BEV特征，提升了模型对动态驾驶环境的预测和理解能力。实验表明，HERMES在生成误差和理解指标上均优于现有技术，展示了其在自主驾驶场景中的有效性。
HERMES的核心方法包括以下几个步骤：
BEV表示：使用鸟瞰视图将多视角图像压缩为统一的BEV表示，保留几何关系。
世界查询：在处理BEV特征时，初始化一组世界查询，这些查询通过大语言模型（LLM）获取世界知识，并与BEV特征进行交互。
当前到未来链接：通过当前到未来的链接模块，利用交叉注意力机制生成未来的BEV特征，确保生成的场景演变与世界知识相结合。
渲染模块：将生成的BEV特征转换为点云，以提供场景的几何信息，为理解和生成任务提供支持。
HERMES通过这些步骤实现了场景理解与生成的无缝结合，克服了传统模型的局限性。
在实验部分，HERMES在多个数据集上进行了全面评估，包括NuScenes和OmniDrive-nuScenes。通过与专注于理解和生成的现有模型进行对比，HERMES在理解任务（如视觉问答和场景描述）上表现出色，CIDEr指标提升了8.0%。在生成任务中，HERMES有效地减少了32.4%的生成误差，显示出其在未来场景预测中的优势。此外，HERMES的训练分为三个阶段，分别为Tokenizer训练、BEV-文本对齐和理解与生成统一，确保了模型的有效性和稳定性。
HERMES的工作原理可以简单理解为一个智能驾驶助手，它不仅能看懂周围的环境，还能预测未来会发生什么。首先，它通过将多个摄像头拍摄到的图像转换成一种鸟瞰图，来获取周围的空间信息。接着，HERMES会提取一些关键问题，像是在问：“前面有什么？”或者“后面有车吗？”这些问题帮助它更好地理解当前的驾驶场景。然后，HERMES通过一种特殊的链接，结合这些理解，预测未来的场景变化，比如前方的路会不会被堵住。最后，它将这些信息转换成点云数据，帮助车辆做出更好的决策。总的来说，HERMES就像一个既能观察又能预测的智能助手，让自动驾驶变得更加安全和高效。
