296-A2.1｜基于流的图像视频生成基础模型，姿势引导DiT人类动作视频生成，DiT高分辨视频生成，资源受限移动设备文生视频

2025-02-07｜HKU,ByteDance｜⭐️🟡
http ://arxiv.org/abs/2502.04896v2https ://saiyan-world.github.io/goku/
本文介绍了Goku，一个先进的联合图像与视频生成模型系列，利用修正流变换器（rectifiedflowTransformer）实现行业领先的性能。Goku的成功依赖于多个关键要素，这些要素包括数据处理管道、模型架构设计、流的公式化以及高效、稳健的大规模训练基础设施。Goku在定性和定量评估中表现卓越，在多个主要任务上设立了新的基准。例如，在文本到图像生成任务中，Goku在GenEval上得分为0.76，在DPG-Bench上得分为83.65，而在文本到视频任务中，Goku在VBench上得分为84.85。这项工作为研究界在联合图像和视频生成模型的开发中提供了宝贵的见解和实用的进展。
Goku模型的设计围绕四个核心组件展开：数据处理、模型架构设计、流的公式化和训练基础设施优化。具体方法如下：
数据处理管道：构建高质量的图像和视频文本数据集，采用视频和图像过滤、OCR驱动的内容分析和主观评估等多种技术，确保视觉和上下文质量。
联合图像视频变分自编码器（VAE）：使用3D联合图像视频VAE，将图像和视频输入压缩到共享潜在空间，以便进行统一表示学习。
Goku变换器架构：基于Transformer设计，采用全注意力机制，允许图像和视频的无缝联合训练，提升生成质量。
流基训练：采用修正流算法，通过线性插值将样本从先验分布逐步转变为目标数据分布，简化建模过程并加快收敛速度。
在实验部分，Goku在多个文本到图像和文本到视频的基准上进行了评估，展示了其竞争优势。在文本到图像生成方面，Goku-T2I在GenEval、T2I-CompBench和DPG-Bench等多个基准上表现出色，特别是在处理详细提示时，Goku-T2I的得分显著高于其他模型。在文本到视频生成方面，Goku-T2V在UCF-101的零-shot生成任务中也取得了优异成绩，展现了在生成视频方面的强大能力。定性结果进一步证明了Goku生成媒体样本的优越质量，强调了其在多模态生成中的有效性及其作为高性能解决方案的潜力。
在Goku的设计中，研究团队采用了一种高效的数据处理方法来确保生成的图像和视频质量。首先，他们收集了大量的图像和视频数据，并使用一系列技术来筛选和过滤这些数据，以确保它们不仅美观而且内容丰富。接下来，他们使用了一种叫做联合图像视频变分自编码器（VAE）的技术，这种技术可以将图像和视频压缩成一种共同的形式，便于计算机理解和处理。然后，Goku的核心是一个基于Transformer的模型，它能够在生成图像和视频时同时考虑文本描述和视觉内容。最后，他们采用了一种叫做流基训练的方法，这种方法通过逐步调整生成的内容，使其更接近真实数据，从而提高了生成的速度和质量。简单来说，Goku通过巧妙的技术组合，使得计算机能够更好地理解和生成丰富的视觉内容。
2025-02-07｜ZJU,ByteDance｜⭐️🟡
http ://arxiv.org/abs/2502.04847v2https ://agnjason.github.io/HumanDiT-page/
HumanDiT是一种基于Pose-GuidedDiffusionTransformer(DiT)的框架，旨在生成高保真度的长格式人类动作视频。尽管人类动作视频生成技术近年来取得了显著进展，但现有方法在细致渲染身体部位（如手和脸）时仍面临挑战，特别是在处理复杂动作和长序列时。HumanDiT通过引入前缀潜在参考策略，支持多种视频分辨率和可变序列长度，从而提升了个性化特征的保持能力。此外，该框架结合了Keypoint-DiT模块，在推理阶段生成后续姿态序列，确保视频续播的流畅性。通过在包含14,000小时高质量视频的大型数据集上进行训练，HumanDiT展示了其在生成高保真、姿态准确的视频方面的优越性能。
HumanDiT的方法主要包括以下几个关键部分：
数据准备：构建一个包含14,000小时人类动作的视频数据集，数据样本包括视频序列、对应的身体姿态序列、背景关键点序列和文本区域。
潜在扩散模型：使用3D变分自编码器（VAE）将视频序列编码为潜在空间，采用扩散模型逐步去噪，生成高质量视频。
前缀潜在参考策略：在生成过程中，第一帧的潜在特征作为参考，确保生成视频与输入图像在视觉特征上的一致性。
姿态引导：引入姿态引导器，以提取空间和时间特征，并通过patchify技术将姿态图像转换为序列标记。
视频续播与姿态转移：通过Keypoint-DiT生成后续姿态，支持从单一图像进行视频续播，并通过姿态适配器实现姿态转移。
在实验部分，HumanDiT通过与多个最先进的方法进行比较，评估其在视频生成中的性能。使用TikTok数据集和自收集的舞蹈和对话场景视频进行定量评估，结果显示HumanDiT在L1损失、峰值信噪比（PSNR）、结构相似性指数（SSIM）等指标上均表现出显著优势。此外，通过用户研究，参与者在时间一致性、身份保持和视觉质量等方面普遍认为HumanDiT优于其他对比方法。实验结果表明，HumanDiT不仅在生成长格式视频时保持高质量，还能有效模拟复杂的人类动作。
HumanDiT的工作原理可以简单理解为一个高级视频生成工具。首先，它收集了大量的人类动作视频，帮助模型学习如何生成新的动作。然后，它利用一种叫做潜在扩散模型的技术，将视频压缩成更小的形式，便于处理。生成视频时，HumanDiT会首先从一个给定的参考图像开始，并使用这个图像的特征来指导后续的动作生成。它还会提取出身体的姿势信息，确保生成的视频在动作上与参考图像相匹配。最后，HumanDiT能够从一帧图像中创造出连续流畅的视频，甚至可以将一种姿势转移到另一种动作上，确保生成的视频既真实又符合预期。
2025-02-07｜HKU,CUHK,ByteDance｜⭐️🟡
http ://arxiv.org/abs/2502.05179v1https ://github.com/FoundationVision/FlashVideo
在高分辨率视频生成领域，DiT扩散模型已取得显著进展，但在内容和运动的保真度上仍面临挑战。为了克服这些问题，研究者们提出了FlashVideo，一个新颖的两阶段框架。该框架旨在通过合理分配模型容量和函数评估次数（NFEs），在生成保真度和质量之间取得平衡。第一阶段专注于低分辨率生成，利用大型模型和足够的NFEs来提高计算效率；而第二阶段则通过流匹配技术在低分辨率和高分辨率之间建立联系，有效生成细节，显著减少NFEs。实验结果表明，FlashVideo在高分辨率视频生成方面达到了最先进的性能，同时保持了出色的计算效率。
FlashVideo采用两阶段的生成流程，首先将视频像素压缩为潜在特征。具体方法如下：
低分辨率阶段（StageI）：
使用5亿参数的CogVideoX-5B模型，生成与输入提示相对应的低分辨率视频（270p）。
通过参数高效微调（PEFT）适应低分辨率，确保计算效率。
该阶段生成的内容和运动与用户输入高度一致，保持语义保真度。
高分辨率阶段（StageII）：
采用2亿参数的CogVideoX-2B模型，专注于细节增强，生成1080p高分辨率视频。
使用流匹配技术将低分辨率潜在表示映射到高分辨率，减少冗余采样步骤，仅需4次函数评估。
通过引入潜在和像素降质，模拟低质量视频，增强模型在高分辨率下的表现。
本研究构建了一个高质量视频数据集，包含200万条1080p视频，并通过美学和运动过滤确保数据质量。实验中，第一阶段模型在270p分辨率下训练50,000次，第二阶段模型则经过多阶段训练以提高性能。通过VBench-Long基准测试，FlashVideo在语义得分和视觉质量上均表现优异。与现有方法相比，FlashVideo不仅在生成速度上有显著优势，还能在用户预览后决定是否进入高分辨率生成阶段，从而降低计算成本和等待时间。
FlashVideo的工作原理可以想象成一个两步走的过程。首先，它像画画一样在较小的画布上（低分辨率）快速绘制出一个大致的图像，确保内容和动作与我们想要的相符。这个过程使用了一个非常强大的工具（大模型），但由于画得较小，所以速度快，耗费的资源少。接下来，第二步就像是用放大镜仔细修整这幅画，增加细节和色彩，让画面看起来更加生动和真实。这一步使用了一个稍小的工具（小模型），但通过聪明的技巧（流匹配），可以在更少的时间内实现更高的画质。这样，用户可以先看到大致的效果，再决定是否投入更多资源来生成更高质量的视频。
2025-02-05｜UNIST｜⭐️🟡
http ://arxiv.org/abs/2502.04363v1https ://github.com/eai-lab/On-device-Sora
在这项研究中，作者提出了On-deviceSora，这是一个创新的解决方案，旨在实现基于扩散模型的文本到视频生成，专为资源有限的移动设备设计。通过在智能手机级设备上高效运行，On-deviceSora克服了传统视频生成模型在计算和内存方面的限制。该系统基于Open-Sora，采用了三种新颖的方法来优化视频生成过程：线性比例跳跃（LPL）、时间维度令牌合并（TDTM）和动态加载的并发推理（CI-DL）。实验结果表明，On-deviceSora在生成高质量视频方面表现出色，能够与高端GPU上的Open-Sora相媲美，从而为普通用户提供了更广泛的生成能力，同时确保了用户隐私和降低了对云基础设施的依赖。
On-deviceSora方法的核心在于解决三个主要挑战：过多的去噪步骤、密集的令牌处理和高内存需求。具体而言：
线性比例跳跃（LPL）：通过利用欧拉法，LPL显著减少了所需的去噪步骤数量。它通过在去噪过程中进行大幅度跳跃，降低了从初始到目标分布的转换时间。
时间维度令牌合并（TDTM）：该方法通过在注意力层中合并连续的时间令牌，减少了计算负担，从而降低了自注意力和交叉注意力的计算复杂度。这一过程能够保持视频质量的同时，显著提高处理效率。
动态加载的并发推理（CI-DL）：为了在内存受限的移动设备上执行大型模型，CI-DL将模型划分为更小的块，并实现并行加载和推理。这种方法有效地提高了内存利用率，并减少了模型推理的延迟。
在实验部分，研究者在iPhone15Pro上实现了On-deviceSora，并进行了广泛的性能评估。通过与运行在NVIDIAA6000GPU上的Open-Sora进行比较，实验结果显示，On-deviceSora能够生成与Open-Sora相当质量的视频，同时显著提升了生成效率。具体来说，使用LPL、TDTM和CI-DL三种方法后，On-deviceSora在去噪步骤和内存使用方面都表现出色，能够在移动设备的限制下实现高质量的视频生成。这些结果表明，On-deviceSora在资源有限的环境中为视频生成开辟了新的可能性。
On-deviceSora的工作原理可以简单理解为三种主要策略的结合，帮助手机生成视频时更加高效。首先，线性比例跳跃就像是快速跳过一些步骤，让视频生成更快完成。其次，时间维度令牌合并则是将连续的帧合并处理，减少了计算量，这样手机在处理视频时不会那么吃力。最后，动态加载的并发推理则是把大模型分成小块，手机可以一边加载新数据一边处理已加载的数据，这样可以同时进行，节省时间。这些方法共同作用，使得即使在普通手机上，用户也能快速生成高质量的视频，而不需要依赖于强大的服务器或云计算。
