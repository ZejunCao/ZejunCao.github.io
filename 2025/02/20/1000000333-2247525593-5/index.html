296-A2.2｜DiT视频生成效率问题（滑动瓦片注意力，缓存与剪枝）；高效高质量长音频生成

主编｜庄才林（CailinZhuang）技术支持｜胡耀淇（YaoqiHu）发布日期｜2025年2月10日｜周一
2025-02-06｜UCSD,UMich,THU,UCBerkeley,MBZUAI｜⭐️🟡
http ://arxiv.org/abs/2502.04507v1
在视频生成领域，DiffusionTransformers（DiTs）凭借其强大的3D全局注意力机制而成为了领先的架构，能够生成高分辨率和长时段的视觉连贯输出。然而，这种全注意力机制的计算成本极高，以至于生成5秒的720p视频时，注意力计算就占据了总推理时间的绝大部分。为了解决这一问题，本文提出了滑动瓦片注意力（SlidingTileAttention,STA），通过利用预训练视频扩散模型中注意力分数的局部化特征，显著减少了计算复杂度。STA通过分块滑动的方式来处理视频帧，优化了计算效率，同时保持了生成的质量，使得生成过程的延迟大幅下降，提升了实际应用的可行性。
滑动瓦片注意力（STA）是一种硬件友好的注意力机制，专门针对视频生成任务进行优化。其核心思想包括：
分块操作：STA将视频分割成不重叠的瓦片，每个瓦片内的查询只关注相同区域内的键，从而确保高效的计算模式。
稀疏注意力管理：通过异步加载数据和计算，STA能够有效管理稀疏注意力掩码，避免了全局掩码计算带来的开销。
头部专业化：不同的注意力头对于局部信息的关注程度不同，STA通过自动配置每个头的窗口大小，平衡了计算效率与生成质量。
训练与微调：STA可以在无需训练的情况下直接替代传统的3D注意力，也可以通过微调进一步提升效率和质量。
在实验部分，研究团队在HunyuanVideo这一最新的开源视频DiT上评估了STA的性能。实验结果显示，STA在生成720p的5秒视频时，推理时间从945秒减少至268秒，且在质量上仅有微小的损失。与其他稀疏注意力方法相比，STA在执行效率和生成质量上都表现出显著优势。通过与不同的基线方法进行比较，STA显示了其在视频生成任务中的高效性，尤其是在处理高分辨率和长时段视频时，极大地提高了计算效率和生成速度。
滑动瓦片注意力（STA）可以被看作是一种更聪明的方式来处理视频生成中的注意力计算。想象一下，当你看一段视频时，实际上你并不需要关注每一帧的每一个细节，很多帧之间的变化是非常小的。STA利用这一点，把视频分成小块（瓦片），而不是处理每一个像素点。这样，当计算每一块时，它只关注块内的内容，而不是整个视频，这样可以节省很多计算资源。此外，STA还会根据不同的注意力头（可以理解为不同的视角）来调整关注的大小，以确保生成的视频既快又好。总之，STA让视频生成变得更高效，减少了等待时间，同时保持了画质。
2025-02-06｜LiAuto,HIT｜⭐️🟡
http ://arxiv.org/abs/2502.04393v1
近年来，扩散变换器（DiT）在视频生成领域取得了显著进展，然而其计算复杂性和资源消耗问题仍然是主要挑战。现有方法虽然利用了相邻扩散步骤之间的注意力差异，但仍然面临突发错误和显著差异的问题。为了解决这些问题，本文提出了UniCP，一个统一的缓存和剪枝框架，旨在提高视频生成的效率。UniCP通过引入错误感知动态缓存窗口（EDCW）、基于主成分分析的切片（PCAS）和动态权重转移（DWS）策略，在时间和空间两个维度上优化了计算效率，同时保持了视频的细节保真度。实验结果表明，UniCP在性能和效率上均优于现有方法，为扩散变换器的应用提供了更为灵活和高效的解决方案。
UniCP的核心方法包括三项主要策略：
错误感知动态缓存窗口（EDCW）：该策略根据实时错误反馈动态调整缓存窗口大小和策略，以应对缓存过程中可能出现的突发错误。具体而言，对于每个注意力块，缓存步骤是根据用户定义的错误阈值和当前注意力输出之间的误差来确定的。
基于主成分分析的切片（PCAS）：在U形错误分布的两端，注意力块的输出可能会显著偏离。在这些情况下，直接缓存可能会导致性能下降，因此PCAS通过修剪冗余的注意力组件，进一步降低计算复杂度。该方法通过压缩查询和键矩阵的维度来实现。
动态权重转移（DWS）：为了统一缓存和剪枝策略，DWS策略通过动态选择在缓存和剪枝之间切换，优化生成过程。它根据生成过程中的缓存图，识别不可缓存的块，选择性地进行剪枝，并存储剪枝前后的权重，以实现更高效的计算。
在实验部分，UniCP被集成到多种最先进的DiT视频生成模型中，如OpenSora、Latte和CogVideoX，以评估其有效性。通过与基线方法（如PAB和FasterCache）进行比较，使用VBench作为评估数据集，实验结果显示，UniCP在不同的错误阈值下表现出稳定的性能，计算复杂度和延迟显著降低，同时保持视频质量。多个评估指标（如LPIPS、SSIM、PSNR）被用于量化生成视频的视觉质量，结果表明，UniCP在加速视频生成的同时，几乎没有牺牲视频质量。
UniCP的工作原理可以简单理解为一个智能的“视频生成助手”，它通过三种方式来提高生成视频的速度和质量。首先，它会根据生成过程中出现的错误情况，自动调整它的工作方式，这就像一个灵活的调度员，根据实时情况来安排工作。其次，当系统发现某些部分的信息冗余时，它会选择性地“削减”这些冗余部分，以减少计算负担，这就像在一个大型项目中，去掉那些不必要的步骤。最后，UniCP会根据当前的工作状态，灵活地在“缓存”和“剪枝”之间切换，确保在处理视频时既快速又高效。这种灵活的处理方式，使得视频生成不仅更快，而且质量依然保持在高水平。
2025-02-07｜USTC,THU,iFlytek｜⭐️🟡
http ://arxiv.org/abs/2502.05130v1https ://swapforward.github.io/
本研究提出了一种新颖的音频生成方法——SwapForward（SaFa），旨在提高长音频生成的质量和效率。传统的扩散模型在生成长音频时面临高计算成本和生成精度低的问题，尤其是在频谱生成和跨视图一致性方面。现有的多视图联合扩散方法虽然提供了一种效率较高的选择，但在处理频谱生成时却存在严重的重叠失真和高一致性成本。为了解决这些问题，SaFa采用了一种框架级的潜在交换策略，通过同步多个扩散过程生成全局一致的长音频，保留更多的频谱细节。SaFa的核心在于双向自环潜在交换，通过相邻视图间的逐步扩散轨迹，适应性地增强高频成分，同时不干扰低频成分，从而实现更高质量的音频生成。
在方法部分，SaFa的设计包含几个关键步骤：
潜在空间的联合扩散：将音频生成视为在潜在空间进行的联合扩散过程，通过合并多个子视图潜在映射，生成一个长的潜在音频图。
自环潜在交换（Self-LoopLatentSwap）：在相邻子视图间应用双向潜在交换操作，使得在重叠区域内能够有效保留高频细节，减少潜在混叠。
参考引导潜在交换（Reference-GuidedLatentSwap）：在非重叠区域引入参考轨迹，引导生成过程以确保跨视图一致性。这一过程通过在前期的去噪步骤中实施，提升了生成音频的整体质量。
逐步差异化轨迹构建：通过分析潜在轨迹的相似性和差异性，设计出步骤化的潜在交换策略，增强生成音频的细节和一致性。
在实验部分，SaFa的性能通过与多种基线方法进行比较来验证，包括MultiDiffusion（MD）、Merge-Attend-Diffuse（MAD）等。实验结果表明，SaFa在音频生成的语义对齐、频谱质量和一致性方面均优于现有方法。具体来说，在长音频生成任务中，SaFa显著降低了频谱失真，并提升了音频的清晰度和细节保留。此外，在全景图像生成任务中，SaFa也展现出较强的适应性和效率。用户研究进一步支持了定量结果，表明参与者普遍认为SaFa生成的音频在质量和一致性上优于其他方法。
在我们的方法中，SwapForward（SaFa）通过几个简单的步骤来生成优质的音频。首先，我们将音频看作是由多个小部分（称为子视图）组成的，然后利用一种叫做“潜在交换”的技术，将这些小部分结合起来，形成一个完整的音频。这就像在拼图时，把每一块拼到一起。接着，我们使用一种特殊的交换方式，确保在拼接时不会丢失音频中的高频细节，比如清脆的音效。为了让不同部分的音频听起来更一致，我们还引入了一个“参考音频”，它就像是一个模版，帮助我们在生成过程中保持一致性。最后，通过比较不同方法的效果，我们发现SaFa能够生成更清晰、更细腻的音频，给人更好的听觉体验。
